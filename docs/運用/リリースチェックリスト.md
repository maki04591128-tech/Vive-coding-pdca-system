## 📖 この文書について

> この文書は、本システム自体をバージョンアップする際の**確認事項と手順**をまとめたチェックリストです。  
> たとえるなら、「新車の出荷前検査リスト」のようなものです。エンジン・ブレーキ・ライト…すべてが正常に動くことを確認してから出荷（リリース）します。

### 🔰 この文書で使われる主要な用語

| 用語 | かんたんな説明 |
|------|-------------|
| **リリース** | 新しいバージョンのシステムを本番環境に反映すること |
| **stg環境** | ステージング環境。本番と同じ構成のテスト環境 |
| **ドライラン** | 実際には反映せず、動作だけを確認するテスト実行 |
| **Trivy（トリビー）** | セキュリティの脆弱性を自動で検出するツール |
| **ロールバック** | 問題が起きたとき、前のバージョンに戻すこと |
| **Go/No-Go** | リリース後に「問題ないので続行（Go）」か「問題あり（No-Go）」を判断すること |

## 1. 概要

バイブコーディングPDCAシステム自体のバージョンアップ時に使用するチェックリスト。リリースはA操作として4/4承認が必要。

---

## 2. リリース前チェック

### ドキュメント
- [ ] リリースノート作成済みか
- [ ] ADR更新済みか（該当する場合）
- [ ] Runbook/Playbook更新済みか（影響がある場合）

### テスト
- [ ] unit test全パスか
- [ ] lint/型チェックパスか
- [ ] stg環境で結合テスト完了か
- [ ] ドライランで1サイクル以上正常完了か
- [ ] 脆弱性スキャン（Trivy）でHIGH/CRITICALゼロか

### バックアップ
- [ ] リリース前バックアップ作成済みか
- [ ] バックアップ整合性確認済みか

## 4. リリース手順

```bash
# 1. prod停止
python -m vibe_pdca.cli stop --env prod --graceful --reason "リリース作業"
# 2. バージョンアップ
git pull origin main && pip install -e . --break-system-packages
# 3. マイグレーション（必要時）
python -m vibe_pdca.cli migrate --env prod
# 4. 設定検証
python -m vibe_pdca.cli config validate --env prod
# 5. 再起動
python -m vibe_pdca.orchestrator start --env prod --project-id <PROJECT_ID>
```

---

## 5. リリース後確認（Go/No-Go）

| 確認項目 | 期待値 | 判定 |
|---------|--------|------|
| システム起動状態 | `running` | □ |
| Discord通知 | 正常受信 | □ |
| 監査ログ書き込み | エラーなし | □ |
| CI結果 | `success` | □ |
| エラーログ | P0/P1なし | □ |

---

## 6. ロールバック判断基準

システム起動不可・監査ログ失敗・A操作フロー不動作・P0/P1発生時は即ロールバック。
