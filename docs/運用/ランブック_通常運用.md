# Runbook — 通常運用手順書

**文書番号**：VCS-OPS-001  
**版**：1.0  
**作成日**：2026-02-27  
**対象システム**：バイブコーディングPDCAシステム  
**根拠**：要件定義書 §26.6、タスク3-11

---

## 1. 前提・ロール

| ロール | 実施できる操作 |
|--------|--------------|
| **Owner** | 全操作（A承認・停止解除・コスト上限変更など） |
| **Maintainer** | 通常停止・再開・モード切替・ドライラン実行 |
| **Reviewer** | 読み取り・コメントのみ |
| **Auditor** | 監査ログ閲覧・エクスポートのみ |

---

## 2. システム起動

### 2.1 初回起動

> **前提**：Web UI から最終到達点が登録済みであること。  
> `--goal-file` はマイグレーション・テスト用途のオプション。

```bash
cp .env.example .env
# ANTHROPIC_API_KEY, GITHUB_APP_KEY, DISCORD_BOT_TOKEN を設定

docker info --format '{{.SecurityOptions}}'
# "name=rootless" が含まれていること

python -m vibe_pdca.orchestrator start \
  --project-id <PROJECT_ID> \
  --env prod
```

### 2.2 起動後確認

- [ ] Discord `pdca-info` に起動通知が届いていること
- [ ] GitHub に Issue / Milestone が作成されていること
- [ ] 監査ログに起動イベントが記録されていること

---

## 3. 通常停止（Graceful Stop）

**ロール：** Owner / Maintainer

```bash
python -m vibe_pdca.cli stop \
  --project-id <PROJECT_ID> \
  --graceful \
  --reason "週次メンテナンスのため"
```

---

## 4. 緊急停止（Emergency Stop）

**ロール：** Owner / Maintainer

```bash
python -m vibe_pdca.cli stop \
  --project-id <PROJECT_ID> \
  --emergency \
  --reason "P0インシデント疑い"
```

> 緊急停止後は `インシデント対応手順書.md` に従いインシデント対応を開始すること。

---

## 5. 再開（Resume）

- 通常再開：Owner / Maintainer
- コスト超過後：**Ownerのみ**
- P0/P1後：**Ownerのみ**（Discord 4/4承認必須）

```bash
python -m vibe_pdca.cli resume \
  --project-id <PROJECT_ID> \
  --reason "メンテナンス完了"
```

---

## 6. A操作の承認フロー

Discord `pdca-critical` チャンネルのスレッド内に届く承認要求に YES / NO / HOLD で回答する。

| 応答 | コマンド | 条件 |
|------|---------|------|
| 承認 | `/approve A-2026-001` | なし |
| 却下 | `/reject A-2026-001 reason:"理由"` | 理由必須 |
| 保留 | `/hold A-2026-001 condition:"条件" deadline:"期限"` | 条件・期限必須 |

12時間以内に4/4承認が揃わない場合は保留。

---

## 7. B操作の確認フロー

B操作はバックアップ後に自動進行。CI設定変更のみペルソナ3承認を追加要求。

---

## 8. モード切替

**ロール：** Owner / Maintainer

### 8.1 運転モード切替

| モード | 推奨場面 | B操作の扱い |
|--------|---------|------------|
| **手動承認** | 初期・重要フェーズ | Owner承認後に進行 |
| **半自動** | 安定運用期 | バックアップ後に自動進行 |
| **全自動** | 低リスク改善フェーズ | バックアップ後に自動進行 |

```bash
python -m vibe_pdca.cli mode set \
  --project-id <PROJECT_ID> \
  --mode manual|semi-auto|full-auto \
  --reason "理由"
```

### 8.2 縮退モード

```bash
python -m vibe_pdca.cli mode set \
  --project-id <PROJECT_ID> \
  --mode degraded|normal \
  --reason "理由"
```

---

## 9. ドライラン実行

```bash
python -m vibe_pdca.cli run \
  --project-id <PROJECT_ID> \
  --dry-run \
  --cycles 1
```

---

## 10. コスト上限の変更

**ロール：** Ownerのみ

```bash
python -m vibe_pdca.cli config set \
  --project-id <PROJECT_ID> \
  --key daily_cost_limit_usd \
  --value 50.00 \
  --reason "集中実装フェーズのため"
```

---

## 11. ログ確認

```bash
python -m vibe_pdca.cli audit-log --last 20
python -m vibe_pdca.cli cost --period today --breakdown model
python -m vibe_pdca.cli status --project-id <PROJECT_ID>
```
