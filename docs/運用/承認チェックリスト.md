# Approval Checklist — 承認観点チェックリスト

**文書番号**：VCS-OPS-003  
**版**：1.0  
**作成日**：2026-02-27  
**根拠**：要件定義書 §17、§18、§26.6、タスク3-11

---

## 1. 概要

A操作は権限拡大・ポリシー緩和・禁止領域変更・例外マージ等。Discord上で4/4承認（全ロール区分から各1名以上）が必要。タイムアウトは12時間。

---

## 2. A操作 承認チェックリスト（共通）

### 2.1 目的・必要性
- [ ] 操作の目的が明確か
- [ ] 代替案が十分に検討されているか（最低2案提示）
- [ ] マイルストーン/DoDとの関連が明確か

### 2.2 影響範囲
- [ ] 影響コンポーネント・サービスが列挙されているか
- [ ] ダウンタイムの有無・時間が明示されているか
- [ ] 他プロジェクトへの波及影響がないか

### 2.3 セキュリティ・権限
- [ ] 権限が最小限か
- [ ] 秘密情報が露出していないか
- [ ] サンドボックス範囲外アクセスがないか

### 2.4 ロールバック
- [ ] ロールバック方針が明示されているか
- [ ] バックアップが作成済みか

### 2.5 コスト
- [ ] コスト見積もりが明示されているか
- [ ] コスト上限を超過しないか

---

## 3. ロール別 追加確認観点

| ロール | 追加確認 |
|--------|---------|
| **Owner** | 最終到達点との整合・ビジネスリスク・コスト上限 |
| **Maintainer** | 日常運用への影響・CI/CDへの影響 |
| **Reviewer** | コード品質・テストカバレッジ・SBOM/脆弱性 |
| **Auditor** | 監査ログ記録の十分性・コンプライアンス要件 |

---

## 4. プロジェクト種別別 追加A操作（ADR-002参照）

> **正式定義**: ADR-002（VCS-ADR-002）を唯一の情報源とする。以下は概要のみ。

| 種別 | 追加A操作例 |
|------|-----------|
| Webアプリ | 本番DBスキーマ変更、認証ロジック変更、外部決済連携 |
| ライブラリ | 公開APIの破壊的変更、メジャーバージョンアップ、ライセンス変更 |
| CLI | コマンド名変更・削除、デフォルト動作変更 |
| インフラ/IaC | 本番リソース削除、IAM権限拡大、ネットワーク設定変更 |
| データパイプライン | 本番データ変換・削除、スキーマ変更、実行スケジュール変更 |
| モバイル | ストア申請、課金ロジック変更 |

---

## 5. NO・HOLD 時の記録要件

### NO（却下）
- [ ] 却下理由（具体的に）
- [ ] 修正・代替案の提案

### HOLD（保留）
- [ ] 保留条件（解除条件）
- [ ] 必要な情報・証拠
- [ ] 期限

---

## 6. B操作（CI設定変更）のペルソナ承認

PM + 書記 + 他1名 = 3承認が必要。
