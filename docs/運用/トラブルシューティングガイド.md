## 🔧 カテゴリ別トラブルシューティング

### 1. インストール・環境構築

#### Q1-1: `pip install -e ".[dev]"` が失敗する

**症状**: `pip install` 時にエラーが出る

**原因と対処**:

| 原因 | 対処法 |
|------|--------|
| Pythonバージョンが古い | `python --version` で確認。**3.12以上** が必要です |
| pip が古い | `pip install --upgrade pip` を実行 |
| 仮想環境が未作成 | `python -m venv .venv && source .venv/bin/activate` を実行 |
| ネットワークエラー | プロキシ設定を確認。`pip install --proxy http://proxy:port ...` |

```bash
# 推奨手順
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install --upgrade pip
pip install -e ".[dev]"
```

#### Q1-2: `flet` パッケージのインストールに失敗する

**症状**: `pip install -e ".[dev,gui]"` でGUI依存パッケージがインストールできない

**対処法**:
- `flet` はオプション依存です。GUI不要なら `pip install -e ".[dev]"` のみで問題ありません
- `flet` をインストールする場合は OS の要件を確認してください（Linux では `libmpv` 等が必要な場合があります）

#### Q1-3: `mypy` が型エラーを報告する

**症状**: `mypy src/` 実行時にエラーが出る

**対処法**:
```bash
# GUI モジュールを除外して実行（推奨）
mypy src/ --exclude 'src/vibe_pdca/gui/'

# types-PyYAML がインストールされているか確認
pip install types-PyYAML>=6.0
```

### 2. LLM接続

#### Q2-1: LLM呼び出しが `401 Unauthorized` で失敗する

**症状**: OpenAI / Anthropic / Google の API 呼び出し時に認証エラー

**対処法**:

1. `.env` ファイルの APIキー設定を確認:
   ```bash
   # .env.example を参考に設定
   cp .env.example .env
   # エディタで .env を開いてAPIキーを設定
   ```

2. APIキーの有効性を確認:
   ```bash
   # OpenAI の場合
   curl -H "Authorization: Bearer $OPENAI_API_KEY" https://api.openai.com/v1/models
   ```

3. キーのスコープ（権限）を確認（利用するモデルへのアクセス権が必要）

#### Q2-2: LLMの応答が遅い・タイムアウトする

**症状**: PDCAサイクル実行時にLLM応答待ちで停止する

**対処法**:

| 対処 | 詳細 |
|------|------|
| タイムアウト設定 | `config/default.yml` のタイムアウト値を確認・調整 |
| モデル変更 | 軽量モデル（`gpt-4o-mini`等）に切り替え |
| ローカルLLM | Ollama等のローカルLLM使用を検討（[ADR-008](../adr/ADR-008_クラウドローカルLLM切替設計.md) 参照） |
| サーキットブレーカー | 連続失敗時の自動切断機能が有効か確認 |

#### Q2-3: ローカルLLM（Ollama）に接続できない

**症状**: `connection refused` エラーが出る

**対処法**:
```bash
# Ollama が起動しているか確認
ollama list

# Ollama を起動
ollama serve

# 使用したいモデルを事前にダウンロード
ollama pull llama3.2

# ベースURLを設定（デフォルト: http://localhost:11434）
# .env に以下を追加
OLLAMA_BASE_URL=http://localhost:11434
```

### 3. テスト実行

#### Q3-1: テストが失敗する

**症状**: `pytest` 実行時に一部のテストが FAIL になる

**対処法**:

```bash
# まず全テストの状況を確認
pytest --tb=short -q --ignore=tests/test_gui.py

# 特定のテストだけ詳しく見る
pytest tests/test_xxx.py -v --tb=long

# 失敗したテストだけ再実行
pytest --lf -v
```

**よくある失敗原因**:

| 原因 | 対処法 |
|------|--------|
| 依存パッケージ不足 | `pip install -e ".[dev]"` を再実行 |
| Python バージョン | 3.12以上が必要 |
| 設定ファイル不足 | `.env.example` を `.env` にコピー |
| パスの問題 | `src` ディレクトリが `PYTHONPATH` に含まれているか確認 |

#### Q3-2: GUIテストがスキップされる

**症状**: `1 skipped` と表示される

**対処法**:
- これは**正常な動作**です。`flet` パッケージが未インストールの場合、GUIテストは自動スキップされます
- GUIテストを実行するには: `pip install -e ".[dev,gui]"` 後に `pytest tests/test_gui.py -v`

#### Q3-3: `ruff check` で大量のエラーが出る

**症状**: Lint チェックで多数の指摘がある

**対処法**:
```bash
# 自動修正可能なものを修正
ruff check src/ tests/ --fix

# フォーマットの修正
ruff format src/ tests/

# 残りの指摘を確認
ruff check src/ tests/
```

### 4. PDCAサイクル実行

#### Q4-1: PDCAサイクルが途中で止まる

**症状**: PLAN→DO の途中で進まなくなる

**対処法**:

1. **ログを確認**: 監査ログ（`audit/`）で最後のアクションを確認
2. **ガバナンス承認待ち**: B/C操作では承認が必要です → 承認チェックリストを確認
3. **停止条件に該当**: 品質スコアやコスト上限に達した可能性 → 設定値を確認
4. **LLMエラー**: LLM接続の問題 → 「2. LLM接続」セクションを参照

#### Q4-2: コスト上限に達してサイクルが停止した

**症状**: コスト管理機能により自動停止された

**対処法**:
- 現在のコストを確認: `GET /api/v1/metrics`
- コスト上限を調整: 設定ファイルの `cost_limit` を変更
- 軽量モデルに切り替えてコスト削減

#### Q4-3: セッションデータが消えた

**症状**: 再起動後にPDCAサイクルの状態が失われた

**対処法**:
- セッション永続化が有効か確認（`engine/session_persistence.py`）
- 永続化先のファイル/ディレクトリのパーミッションを確認
- バックアップからの復元: `engine/backup.py` のロールバック機能を使用

### 5. CI/CD

#### Q5-1: GitHub Actions が失敗する

**症状**: CI パイプラインの一部ジョブが失敗

**対処法**:

| ジョブ | 失敗時の対処 |
|--------|------------|
| **lint** | `ruff check src/ tests/` をローカルで実行して修正 |
| **type-check** | `mypy src/ --exclude 'src/vibe_pdca/gui/'` を実行して修正 |
| **test** | `pytest --tb=short -q --ignore=tests/test_gui.py` で再現確認 |
| **secrets-scan** | `.gitleaks.toml` の設定確認、誤検知なら除外リストに追加 |

#### Q5-2: CI アダプターの接続テストが失敗する

**症状**: `GitHubActionsAdapter.get_status()` でエラー

**対処法**:
- `GITHUB_TOKEN` 環境変数が設定されているか確認
- トークンに `repo` スコープの権限があるか確認
- API レート制限に引っかかっていないか確認

### 6. Docker・サンドボックス

#### Q6-1: Docker コンテナが起動しない

**症状**: `docker build` または `docker run` が失敗

**対処法**:
```bash
# Docker デーモンが起動しているか確認
docker info

# イメージのビルド
cd docker/
docker build -f Dockerfile.sandbox -t vibe-pdca-sandbox .

# コンテナの実行（ログ付き）
docker run --rm -it vibe-pdca-sandbox
```
