### そもそも「AI推論サーバー」とは？

**AI推論サーバー**とは、ChatGPTのようなAI（人工知能）を自分のコンピュータで動かすための**高性能な専用マシン**のことです。

通常、ChatGPTやClaudeなどのAIは、企業のインターネット上にあるサーバーで動いており、利用するたびに料金がかかります。AI推論サーバーを自前で用意すると、**インターネットを経由せずにAIを動かせる**ようになり、以下のメリットがあります：

| 項目 | クラウドAI（ChatGPT等） | 自前のAI推論サーバー |
|------|----------------------|-------------------|
| 利用料金 | 使うたびにかかる（従量課金） | 電気代のみ（初期投資は必要） |
| データの安全性 | データが外部に送信される | データが自分のサーバー内で完結 |
| 速度の安定性 | 混雑時に遅くなることがある | 常に安定した速度 |
| インターネット接続 | 必須 | 不要（オフラインでも動作） |

## 1. ラックマウントサーバーの基礎

> 📖 **この章で学べること**：サーバーとは何か、どんな種類があるか、型番の読み方と接続端子の基礎

### 1.1 ラックとは

「ラック」とは、サーバーや通信機器を縦に積み重ねて収納する金属製のフレームです。  
家庭にたとえると、本棚のようなものです。規格化されたサイズ（19インチ幅）で統一されており、どのメーカーの機器も同じラックに収まります。

```
┌─────────────────────────┐
│  19インチ幅（標準規格）  │  ← すべての機器がこの幅に合わせて設計
│  ┌───────────────────┐  │
│  │  1U サーバー       │  │  ← 1U = 44.45mm（約1.75インチ）の高さ
│  ├───────────────────┤  │
│  │  2U サーバー       │  │
│  │                   │  │  ← 高さが1Uの2倍
│  ├───────────────────┤  │
│  │  スイッチ / UPS    │  │
│  └───────────────────┘  │
└─────────────────────────┘
```

### 1.2 Uサイズ（Unit）の意味

サーバーの「高さ」を表す単位です。1U = **44.45mm**。

| サイズ | 高さ | 特徴 | 向いている用途 |
|--------|------|------|--------------|
| **1U** | 約44mm | 薄型・省スペース。冷却ファンが小型で爆音になりやすい | ウェブサーバー・ロードバランサー |
| **2U** | 約89mm | バランスが良い。GPU 1〜3枚搭載可能 | **汎用・このシステムに最適** |
| **4U** | 約178mm | 拡張性が高い。GPU 4枚以上搭載可能 | 大規模AI推論・ストレージサーバー |

> **💡 ポイント**: GPUを搭載したい場合は最低 **2U以上** が必要です。1Uは物理的にGPUが入りません。

### 1.3 主要メーカーと型番の読み方

**DELL PowerEdge シリーズ**（最もポピュラー）

```
R 7 4 0 x d
│ │ │ │ │ └── d = ドライブベイ追加版
│ │ │ │ └──── x = GPU拡張スロット追加版
│ │ │ └────── 0 = 世代（0 = 第14世代）
│ │ └──────── 4 = GPUソケット数（4 = デュアルCPU対応）
│ └────────── 7 = シリーズ（7 = エンタープライズ向け）
└──────────── R = ラックマウント型
```

**HP ProLiant DL シリーズ**

```
DL 3 8 0 Gen10
│  │ │ │   └── 世代
│  │ │ └────── 0 = 標準版
│  │ └──────── 8 = デュアルCPU対応
│  └────────── 3 = ミッドレンジ
└──────────── DL = デュアル（2ソケット）ラック
```

### 1.4 接続インターフェースの基礎

| インターフェース | 用途 | 速度 |
|--------------|------|------|
| **PCIe 4.0 x16** | GPUの接続スロット（最重要） | 64GB/s |
| **PCIe 3.0 x16** | 旧世代GPUの接続スロット | 32GB/s |
| **NVMe M.2** | 高速SSDの接続 | 最大7GB/s |
| **SATA III** | 通常SSD・HDDの接続 | 最大600MB/s |
| **1GbE / 10GbE** | ネットワーク接続 | 1〜10Gbps |
| **iDRAC / iLO** | リモート管理用インターフェース（DELL/HP） | — |

> **💡 iDRACとは**: サーバーの電源が切れていてもリモートからOS操作・電源ON/OFFができる管理チップです。自宅サーバーの運用では非常に便利です。

> 📖 **この章で学べること**：バイブコーディングPDCAシステムを動かすために必要な部品と性能の目安

### 2.1 システム特性の整理

バイブコーディングPDCAシステムは以下の特性を持ちます。  
左側が「AIがやること」、右側が「そのために必要な部品・性能」です。

```
特性                          → 必要なリソース
──────────────────────────────────────────────────
5ペルソナ並列LLM推論           → VRAM 大容量 / GPU性能
Dockerコンテナを多数起動        → CPU コア数 / RAM容量
PDCAサイクル最大6時間連続       → 24時間安定稼働
マルチプロジェクト物理分離       → リソース分割余裕
rootlessDockerサンドボックス    → CPUコア・RAM余裕
監査ログ・状態ファイル大量保存   → SSD大容量
```

### 2.2 推奨スペック一覧

| パーツ | 最小構成 | 推奨構成 | 理由 |
|--------|---------|---------|------|
| **CPU** | Xeon E5-2620v4（8コア） | Xeon Gold 6226R × 2（32コア計） | Docker並列実行のコア数確保 |
| **RAM** | 64GB ECC DDR4 | 128GB ECC DDR4 | ペルソナ5体分のコンテキスト保持 |
| **システムSSD** | 500GB NVMe | 1TB NVMe | OS + Dockerイメージ |
| **データSSD** | 1TB SATA SSD | 2TB NVMe | 監査ログ・モデルファイル |
| **GPU** | A30 × 1（24GB VRAM） | **A100 40GB PCIe × 2** | ローカルLLM推論（後述） |
| **NIC** | 1GbE | 10GbE | GitHub API・vLLM通信 |
| **電源** | 750W | 1200W 冗長（2系統） | GPU追加で消費電力増大 |

> **💡 ECCメモリとは**: メモリのビット化けを自動修正する機能付きのRAMです。サーバーでは標準採用されており、長時間稼働の信頼性に直結します。コンシューマー向けPCのDDR4とは別物です。

## 3. サーバー用GPUの基礎知識

> 📖 **この章で学べること**：GPU（AIの「頭脳」にあたるパーツ）の選び方と、AIモデルの大きさとの関係

### 3.1 なぜサーバー用GPUが必要か

> 💡 **GPU（ジーピーユー）とは？**  
> Graphics Processing Unit の略で、もともとは映像処理用のパーツですが、大量の計算を並列で処理できる特性から、AI の処理にも使われるようになりました。  
> 家庭用のパソコンにもGPUが入っていますが、サーバー用GPUは24時間365日の連続稼働を前提に設計されており、信頼性が格段に高くなっています。

| 比較項目 | コンシューマーGPU（RTX系） | サーバーGPU（A100等） |
|---------|------------------------|-------------------|
| **ECCメモリ** | なし | **あり**（長時間推論の信頼性） |
| **耐久性** | 家庭用（数年） | 企業用（5〜7年連続稼働想定） |
| **冷却方式** | 軸流ファン（ラックのエアフローと相性悪い） | **ブロワー型**（前→後に空気を流す） |
| **フォームファクタ** | ダブルスロット（PCIeに飛び出す） | **シングル/フルハイト**（ラックに収まる） |
| **保証** | 基本なし | 3年保証付き製品が多い |
| **NVLink/NVSwitch** | 非対応が多い | **対応**（マルチGPUで高速通信） |

### 3.2 VRAMとモデルサイズの関係

> 💡 **VRAM（ブイラム）とは？**  
> GPU専用のメモリ（作業用の記憶領域）のことです。AIモデル（AIの「知識データ」）はこのVRAMに丸ごと読み込む必要があります。VRAMが大きいほど、大きなAIモデルを動かせます。

ローカルLLM（自前で動かすAI）を使うには **VRAMにモデル全体が収まる** 必要があります。

```
モデルパラメータ数 × 精度のバイト数 ≒ 必要VRAM（概算）

例：Qwen2.5-72B を Q4量子化（4bit）で動かす場合
  72,000,000,000 × 0.5バイト ≒ 36GB VRAM が必要

精度別のバイト数：
  fp32 (32bit)  = 4バイト/パラメータ  ← 訓練時
  fp16 (16bit)  = 2バイト/パラメータ  ← 標準推論
  bf16 (16bit)  = 2バイト/パラメータ  ← A100推奨
  Q8  ( 8bit)   = 1バイト/パラメータ
  Q4  ( 4bit)   = 0.5バイト/パラメータ ← 最小・実用的
```

#### モデル別の必要VRAM早見表

| モデルサイズ | fp16 | Q4量子化 | 実用モデル例 |
|------------|------|---------|------------|
| 7B〜8B | 14GB | **5GB** | Llama3.1-8B, Qwen2.5-7B |
| 13B〜14B | 26GB | **8GB** | Qwen2.5-14B, Phi-4 |
| 30B〜34B | 64GB | **20GB** | Qwen2.5-32B |
| 70B〜72B | 140GB | **38GB** | Llama3.3-70B, **Qwen2.5-72B** |

### 3.3 重要なGPUスペックの読み方

```
NVIDIA A100 PCIe 40GB のスペックシート例

VRAM容量:        40GB HBM2e   ← モデルがここに乗る
メモリ帯域幅:    1,555 GB/s   ← トークン生成速度に直結
FP16 TFLOPS:    77.6 TFLOPS  ← 推論の演算性能
BF16 TFLOPS:    77.6 TFLOPS  ← A100はBF16ネイティブ対応（重要）
TDP:            250W          ← 消費電力
フォームファクタ: PCIe          ← ラックサーバーに直接挿せる
ECC:            あり（HBM2e）  ← 長時間稼働の信頼性
NVLink:         あり           ← 2枚以上で高速連携
```

> **💡 BF16とは**: Brain Floating Point 16bitの略。GoogleのTPUで開発された精度形式で、fp16より数値の表現範囲が広くLLM推論に向いています。A100はBF16を**ハードウェアネイティブ**でサポートしており、量子化なしで高品質な推論が可能です。

### 3.4 NVLinkとテンソル並列の仕組み

複数のGPUに1つの大きなモデルを分割して乗せる技術を**テンソル並列**と呼びます。

```
【テンソル並列の例: A100 40GB × 2枚でQwen2.5-72B(fp16)を動かす】

  GPU 0 (40GB)               GPU 1 (40GB)
  ┌──────────────────┐       ┌──────────────────┐
  │ レイヤー 0〜39    │◄─────►│ レイヤー 40〜79   │
  │ (モデルの前半)    │NVLink│ (モデルの後半)    │
  └──────────────────┘       └──────────────────┘
         ↓                          ↓
    合計80GB相当のモデルを1つとして扱える
```

**vLLMでの設定例**:

テンソル並列を使う場合、vLLMの起動時に `--tensor-parallel-size 2` オプションを指定して「2枚のGPUに分割する」と設定します。

> 📌 **vLLMの具体的な起動手順** → [02_システムセットアップ手順書](../手順書/02_システムセットアップ手順書.md)

### 4.2 用途別推奨

```
【検証・入門】  A30（24GB）× 1   ── 32Bモデル(Q4)が動く。低コストで始める
【本番・推奨】  A100 40GB × 2    ── 72Bモデル(BF16)が動く。このシステムの最適解
【将来拡張】   A100 80GB × 2    ── 量子化なし70B + 複数モデル同時ロード
```

### 4.3 **最終推奨: A100 40GB PCIe × 2枚**

この選定の根拠を整理します。

| 観点 | 理由 |
|------|------|
| **VRAMの十分性** | 40GB × 2 = 80GBで72B(BF16)が量子化なしで動作 |
| **5ペルソナ対応** | テンソル並列で1モデルを2枚に分散。高速推論 |
| **マルチプロジェクト** | GPU 2枚をプロジェクト単位に論理分割可能 |
| **縮退モード継続** | クラウドLLMが障害でも推論継続（§8.2要件） |
| **将来性** | 80GBに換装してもドライバー・ソフトウェアが同一 |
| **コスト** | 50〜90万円/枚。サーバーGPUとして現実的な価格 |

## 6. 電力・冷却・設置環境

> 📖 **この章で学べること**：サーバーの電気代・コンセントの要件・冷却方法・騒音対策

### 6.1 消費電力の計算

GPU搭載サーバーは**消費電力が大きい**ため、事前の計算が必須です。  
家庭用のパソコンが100〜300W程度なのに対し、AI推論サーバーは**約1,000W**（電子レンジに近い消費電力）を使います。

```
【推奨構成（A100 × 2）の消費電力内訳】

  Xeon CPU × 2:          ≒ 300W
  A100 40GB × 2:         ≒ 500W（TDP 250W × 2）
  RAM 128GB:             ≒  50W
  NVMe SSD × 2:          ≒  20W
  マザーボード・ファン:    ≒  80W
  ────────────────────────────────
  合計（通常時）:         ≒ 950W
  合計（ピーク時）:       ≒ 1,100W

  → 電源ユニットは 1,200W 以上を選択
  → 電気代：月 約10,000〜12,000円（24時間稼働、28円/kWh計算）
```

### 6.2 電源コンセントの要件

```
家庭用100V コンセント（15A）
  → 最大1,500W まで対応可能
  → 推奨構成は厳しい（ピーク時に近い）

家庭用200V コンセント（20A）← 推奨
  → 最大4,000W まで対応可能
  → 余裕を持って運用可能
  → エアコン用の既存コンセントが流用できる場合がある
```

> **⚠️ 注意**: 電気工事士の免許が必要なため、200Vコンセントの増設は電気工事業者への依頼が必要です（工事費：3〜5万円程度）。

### 6.3 UPS（無停電電源装置）の重要性

> 💡 **UPS（ユーピーエス）とは？**  
> Uninterruptible Power Supply の略で、停電が起きたときに一時的にバッテリーから電気を供給する装置です。  
> パソコンの「モバイルバッテリー」のようなものだと考えてください。サーバーを安全にシャットダウン（終了）するための時間を稼ぎます。

PDCAサイクルが最大6時間走るため、**停電やブレーカー落ちでの強制終了は致命的**です。

```
UPS の選定目安

  バッテリー保持時間の計算:
    UPS容量(VA) × 力率(0.6) ÷ サーバー消費電力(W) × 60 ≒ 保持時間(分)

  例: APC Smart-UPS 3000VA（推奨）
    3,000 × 0.6 ÷ 1,000 = 1.8 → 約10〜15分のバッテリー保持

  → 目的は「正常なシャットダウンの時間を稼ぐ」こと
  → 長時間の停電には対応しない（それはデータセンターの話）
```

| 製品 | 容量 | 想定保持時間 | 価格目安 |
|------|------|-----------|---------|
| APC Smart-UPS 1500VA | 1,500VA | ≒ 5分 | 約5万円 |
| **APC Smart-UPS 3000VA** | 3,000VA | **≒ 15分** | 約15万円 |
| APC Smart-UPS 5000VA | 5,000VA | ≒ 25分 | 約30万円 |

### 6.4 冷却（エアフロー）の基礎

ラックサーバーは**前面から吸気・背面に排気**する設計です。

```
【正しいエアフロー】

  前面（吸気）    ──────────────────────→    背面（排気）
  冷たい空気      CPU / メモリ / GPU          熱い空気
                  ↑ 熱源はここに集中

【自宅での設置注意点】
  ❌ 背面を壁に密着させる  → 排気した熱風を再吸気してしまう
  ✅ 背面から 30cm 以上 離す
  ✅ エアコンで室温を 25℃ 以下に保つ
  ✅ 夏場は特に注意（GPUはサーマルスロットリングで性能低下）
```

### 6.5 騒音について

サーバーは**非常にうるさい**です。事前に覚悟が必要です。

| サーバータイプ | 騒音レベル | 比較 |
|-------------|---------|------|
| 1U サーバー | 70〜80dB | 掃除機と同程度 |
| **2U サーバー** | **55〜70dB** | エアコン室外機〜掃除機 |
| タワーサーバー | 45〜60dB | 比較的静か |

**対策案:**
- 防音ラックボックスを使用（20〜50万円）
- 別室（クローゼット・納戸）に設置
- サーバー室専用空間を確保

## 7. ソフトウェアスタック構成

> 📖 **この章で学べること**：サーバーにインストールするソフトウェアの全体像と、各ソフトの役割
>
> 💡 **ソフトウェアスタックとは？**  
> 「スタック」は「積み重ね」という意味です。OS（基本ソフト）の上に、必要なソフトウェアを順番に積み重ねて構築します。

### 7.1 全体構成図

> 💡 以下の図は、サーバーにインストールするソフトウェアの層構造です。下から順にインストールしていきます。

```
Ubuntu 22.04 LTS（OS）← サーバーの基本ソフト（Windowsの代わり。無料で使える）
│
├── CUDA 12.x + cuDNN 9.x（GPUドライバー）
│   └── GPUをOSから操作するための必須ソフト（GPUの「通訳」のような存在）
│
├── Docker Engine（rootlessモード）← ADR-007 要件
│   └── プログラムを隔離された安全な空間（コンテナ）で実行するための仕組み
│
├── vLLM（ローカルLLM推論サーバー）
│   └── AIモデルを動かすためのサーバーソフト。ChatGPTと同じ形式のAPI（通信窓口）を提供
│       → 既存のシステムに無改変で接続可能
│
├── バイブコーディングPDCAシステム（Python）
│   └── 本システムの本体。AIへの指示出しとPDCAサイクルの管理を行う
│
├── Prometheus + Grafana（監視）← §14 可観測性要件
│   └── サーバーの状態（GPU温度・メモリ使用率など）をグラフで見える化するツール
│
└── UFW / nftables（ファイアウォール）← ADR-007 要件
    └── 不正アクセスからサーバーを守る「門番」。許可した通信先のみ接続を許可
```

### 7.2 インストールの流れ

ソフトウェアのインストールは、下の層から順番に行います。各ステップの**目的**を理解してから手順書に進んでください。

| 順番 | インストール対象 | 目的 | たとえるなら |
|:----:|----------------|------|------------|
| 1 | Ubuntu 22.04 LTS（OS） | サーバーの基本ソフトを入れる | パソコンにWindowsを入れるのと同じ |
| 2 | NVIDIAドライバー + CUDA | GPUをOSから操作できるようにする | GPUの「通訳ソフト」を入れる |
| 3 | rootless Docker | プログラムを安全な隔離空間で実行できるようにする | プログラム用の「安全な個室」を作る仕組み |
| 4 | vLLM | AIモデルを動かすサーバーソフトを入れる | AIの「エンジン」を起動する |
| 5 | PDCAシステム本体 | 本システムをインストールする | 最終的に使うアプリを入れる |

> 📌 **ステップ1〜2の具体的な手順** → [01_ハードウェア構築手順書](../手順書/01_ハードウェア構築手順書.md)  
> 📌 **ステップ3〜5の具体的な手順** → [02_システムセットアップ手順書](../手順書/02_システムセットアップ手順書.md)  
> 📌 **ローカルPCで行う場合** → [03_ローカルセットアップ手順書](../手順書/03_ローカルセットアップ手順書.md)

### 7.3 GPU監視の概要

サーバー稼働後は、GPUの状態を常に監視する必要があります。主に以下の項目をチェックします。

| 監視項目 | 正常範囲 | 異常時の影響 |
|---------|---------|------------|
| **GPU温度** | 80℃以下 | 高温になるとサーマルスロットリング（自動的に性能を下げる保護機能）が発動 |
| **VRAM使用量** | モデルサイズ以内 | 超過するとモデルが読み込めない |
| **GPU使用率** | 推論中は80〜100% | 低すぎるとボトルネックが他にある可能性 |
| **消費電力** | TDP以内 | 電源容量の超過は故障の原因に |

> 📌 **GPU監視の具体的なコマンドと手順** → [04_運用保守手順書 §7](../手順書/04_運用保守手順書.md#7-監視とアラート対応)  
> 📌 **ハードウェア保守** → [04_運用保守手順書 §13](../手順書/04_運用保守手順書.md#13-ハードウェア保守)

## 8. 格安購入先・中古市場

> 📖 **この章で学べること**：サーバー本体とGPUをどこで安く購入できるか
>
> 💡 サーバーやGPUは「中古品」が非常に多く流通しています。企業のデータセンターで使われていたものが放出されるためです。  
> 新品の半額以下で購入できることも多く、性能は新品と同等です。

### 8.1 サーバー本体の購入先

| 購入先 | 特徴 | 価格帯 | リスク |
|--------|------|--------|--------|
| **ヤフオク** | 最安値。DELLのデータセンター放出品が多い | 5〜30万円 | 保証なし。動作確認が重要 |
| **じゃんぱら** | 動作確認済み。店舗で現物確認可能 | 10〜50万円 | 低い |
| **メルカリ** | 個人出品。値段交渉可 | 3〜20万円 | やや高い |
| **DELL アウトレット** | 整備済み品。部分的に保証付き | 20〜80万円 | 低い |
| **Bargain Hardware (UK)** | 整備済み・保証付き。英国から輸入 | 15〜60万円 | 低い（送料・関税注意） |

**ヤフオクでの狙い目ワード:**
```
"PowerEdge R740" "PowerEdge R730" "Xeon Gold" "ECC DDR4"
→ 入札数が少ない深夜終了のオークションが狙い目
```

### 8.2 サーバーGPUの購入先

| 購入先 | 特徴 | 狙い目 |
|--------|------|--------|
| **eBay（米国）** | A100のデータセンター放出品が世界最安。英語対応必要 | A100 40GB PCIe |
| **ヤフオク** | 国内最安帯。A30・V100が流通している | A30, A10G |
| **秋葉原 ツクモ / SOFMAP** | 動作確認済み。RTX系は豊富だがA100は少ない | A30, A10G |
| **Bargain Hardware (UK)** | A100・L40Sの整備済み品。日本への発送可 | A100 40GB |
| **Alibaba（上級者向け）** | 中国DCの大量放出品。最安だが品質リスクあり | 上級者のみ |

#### eBayでのA100購入ガイド

```
検索キーワード: "NVIDIA A100 40GB PCIe" site:ebay.com

注意事項:
  ✅ "Tested and Working" の記載があるものを選ぶ
  ✅ フィードバック評価 98%以上のセラーを選ぶ
  ✅ 輸入消費税（商品価格の10%）と関税が別途かかる
  ✅ 送料は $50〜$150 程度
  ⚠️ "For parts" と書かれたものは故障品の可能性が高い
  ⚠️ SXM版（サーバーノード専用）とPCIe版を間違えない
```

> **💡 SXMとPCIeの違い**: A100には「SXM」と「PCIe」の2種類があります。SXM版はNVIDIA DGXなどの専用サーバーにしか搭載できない特殊コネクタです。一般のラックサーバーには必ず**PCIe版**を購入してください。

### 8.3 購入前の確認チェックリスト

```
サーバー本体:
  □ 電源ユニット（PSU）が2系統（冗長電源）か
  □ PCIeスロットの空きが十分か（GPU枚数分）
  □ 補助電源コネクタ（8pin / 6pin）が付属しているか
  □ iDRAC（リモート管理）ライセンスが有効か
  □ レールキット（ラック固定用）が付属しているか

サーバーGPU:
  □ PCIe版（SXM版ではない）か
  □ フルハイト・フルレングスか（サーバー用）
  □ ブロワー型冷却（Passive / Active）か
  □ ECC搭載と記載があるか
  □ 出品者の動作確認の具体的な証拠（nvidia-smiのスクリーンショット等）があるか
```

### フェーズ1: 検証（〜3ヶ月）目安予算 50万円

> 💡 まず少ない投資で「本当にローカルAIで動くか」を確認するフェーズです。

```
目的: ローカルLLMが実際にこのシステムで動くか確認する

構成:
  DELL R730 中古  +  NVIDIA A30 × 1

作業:
  1. Ubuntu 22.04 LTS インストール
  2. CUDA + Dockerセットアップ
  3. vLLM で Qwen2.5-32B を起動
  4. LLMゲートウェイのフォールバックに追加
  5. PDCAサイクルを1回ローカルLLMで通す
  6. 推論速度・安定性・VRAM使用量を計測

判断基準:
  → 1サイクル（6時間以内）が正常完了すれば次フェーズへ
```

### フェーズ2: 本番化（〜6ヶ月）目安予算 200万円

```
目的: 72Bモデルで全役割をローカルLLMで賄う

構成:
  DELL R740xd 中古  +  NVIDIA A100 40GB PCIe × 2

作業:
  1. フェーズ1の知見でvLLM設定を最適化
  2. Qwen2.5-72B（BF16）をテンソル並列で起動
  3. クラウドLLMをフォールバック専用に降格
  4. Prometheus + Grafana で GPU監視ダッシュボード構築
  5. マルチプロジェクト（§26.1）でGPUリソース分割テスト

判断基準:
  → コスト削減率とレイテンシが許容範囲か確認
```

### フェーズ3: スケールアウト（必要に応じて）

```
目的: 同時並列プロジェクト数の増加・高速化

選択肢A: A100 80GB PCIe × 2 に換装
  → 量子化なしの高品質推論 + 複数モデル同時ロード

選択肢B: A100 40GB × 4枚構成
  → より多くのプロジェクトを並列処理

選択肢C: H100 PCIe × 1
  → 単枚で最高性能。A100の約3倍の推論速度
```

## 10. 用語集

> 📖 この説明書に登場する専門用語をまとめました。わからない用語があったらここを参照してください。

| 用語 | 説明 | たとえるなら |
|------|------|------------|
| **U（Unit）** | ラックの高さ単位。1U = 44.45mm | 本棚の段の高さのようなもの |
| **PCIe（ピーシーアイエクスプレス）** | GPUやSSDをマザーボードに接続する高速バス規格 | パーツを差し込む「コンセント」のようなもの |
| **ECC（イーシーシー）** | エラー訂正コード。メモリのビット化けを自動修正する機能 | メモリの「自動校正機能」 |
| **VRAM（ブイラム）** | GPU専用のビデオメモリ。LLMモデルはここに収まる必要がある | AIの「作業机」。広いほど大きなモデルが動く |
| **TDP（ティーディーピー）** | 熱設計電力。GPUの最大消費電力の目安 | GPUの「最大電力消費量」の目安 |
| **NVLink（エヌブイリンク）** | NVIDIA独自のGPU間高速通信バス。PCIeより約10倍高速 | GPU同士をつなぐ「高速道路」 |
| **テンソル並列** | 1つのAIモデルを複数のGPUに分割して同時に処理する技術 | 大きな荷物を2人で分担して運ぶイメージ |
| **BF16（ビーエフ16）** | Brain Float 16。LLM推論に適した16bit浮動小数点数形式 | AIの「計算精度」の一種。速度と品質のバランスが良い |
| **Q4量子化** | モデルのパラメータを4bitに圧縮する手法。精度をやや犠牲にVRAMを節約 | 高画質の画像を圧縮して容量を減らすようなもの |
| **vLLM（ブイエルエルエム）** | 高性能なLLM推論サーバー。OpenAI互換APIを提供 | AIを動かす「エンジン」ソフトウェア |
| **SXM（エスエックスエム）** | A100の特殊フォームファクタ。専用ノード（DGX等）専用。一般サーバー不可 | 特殊な形状のGPU。普通のサーバーには使えない |
| **iDRAC（アイドラック）** | DELLのリモート管理チップ。電源OFF状態でもリモート操作可能 | サーバーの「リモコン」。遠隔から電源ON/OFFやOS操作ができる |
| **UPS（ユーピーエス）** | 無停電電源装置。停電時に一時的にバッテリー電源を供給する | サーバーの「モバイルバッテリー」 |
| **rootless Docker** | rootユーザー権限なしでDockerを実行するセキュリティモード | 管理者権限なしでも安全にプログラムを隔離実行できる仕組み |
| **Docker（ドッカー）** | プログラムを隔離された安全な空間（コンテナ）で実行するための仕組み | プログラムの「隔離ルーム」を作るツール |
| **コンテナ** | Dockerで作られた隔離されたプログラム実行環境 | アプリを動かす「小さな仮想パソコン」 |
| **ブロワー型冷却** | GPU前面から背面へ排気するサーバー向け冷却方式 | エアコンの送風のような冷却方法 |
| **CUDA（クーダ）** | NVIDIAのGPU向け並列演算プラットフォーム。LLM実行に必須 | GPUを「AI用に使う」ためのソフトウェア基盤 |
| **サーマルスロットリング** | 高温時にGPUが自動的に性能を下げる保護機能 | 「熱くなりすぎたら自動で休む」安全装置 |
| **LLM（エルエルエム）** | Large Language Model。ChatGPTのような大規模言語AI | 文章を理解し、文章で回答するAI |
| **API（エーピーアイ）** | Application Programming Interface。ソフトウェア同士が通信する窓口 | ソフトウェアの「注文窓口」 |
| **OS（オーエス）** | Operating System。コンピュータの基本ソフト（Windows, macOS, Ubuntuなど） | コンピュータの「基礎」となるソフト |
| **Ubuntu（ウブントゥ）** | 無料で使えるLinux系のOS。サーバー用途で世界的に最も人気 | サーバー向けの無料OS |
