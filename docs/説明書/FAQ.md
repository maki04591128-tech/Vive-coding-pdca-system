## 🌟 システム概要

### Q. このシステムは何をするものですか？

**A.** LLM（大規模言語モデル）を使って、ソフトウェア開発を**自動的にPDCAサイクルで回す**システムです。

1. **Plan（計画）**: LLMがゴールを分析し、実装計画を立てる
2. **Do（実行）**: LLMがコードを生成・実装する
3. **Check（検証）**: 5ペルソナ（PM/書記/プログラマ/デザイナ/ユーザ）が自動レビュー
4. **Act（改善）**: レビュー結果に基づいて改善方針を決定

このサイクルを**自動で繰り返す**ことで、高品質なソフトウェアを効率的に開発します。

### Q. どのLLMに対応していますか？

**A.** 以下のLLMプロバイダに対応しています：

| プロバイダ | モデル例 | 備考 |
|-----------|---------|------|
| **OpenAI** | GPT-4o, GPT-4o-mini | クラウド利用 |
| **Anthropic** | Claude 3.5 Sonnet | クラウド利用 |
| **Google** | Gemini Pro | `pip install -e ".[google]"` が必要 |
| **ローカルLLM** | Llama, Mistral等（Ollama経由） | オフライン利用可能 |

複数プロバイダの**自動切り替え**（フォールバック・サーキットブレーカー）にも対応しています。

### Q. ローカルLLMだけで動かせますか？

**A.** はい。[Ollama](https://ollama.com/) を使用すれば、**インターネット接続なし**で完全にローカル環境で動作します。

```bash
# Ollamaをインストール後、役割別モデルをダウンロード
ollama pull codestral:22b   # まず軽量モデルで動作確認（12GB程度）

# .env に設定
VIBE_PDCA_LLM_MODE=local
```

> **🚀 高性能モデル**: VRAM 48GB 以上の環境では `deepseek-v3` や `devstral:24b` なども利用可能です。  
> 詳細は [ローカルセットアップ手順書 §6.3](../手順書/03_ローカルセットアップ手順書.md#63-役割別モデルのダウンロード) を参照してください。

詳細は [ADR-008 クラウド/ローカルLLM切替設計](../adr/ADR-008_クラウドローカルLLM切替設計.md) を参照してください。

## 🛠️ セットアップ

### Q. 動作環境は何が必要ですか？

**A.** 最小要件は以下のとおりです：

| 項目 | 要件 |
|------|------|
| **Python** | 3.12 以上 |
| **OS** | Linux / macOS / Windows（WSL2推奨） |
| **メモリ** | 4GB以上（ローカルLLM使用時は16GB以上推奨） |
| **ディスク** | 500MB以上（ローカルLLMモデル含まず） |

### Q. インストール手順を教えてください

**A.**

```bash
# 1. リポジトリのクローン
git clone https://github.com/maki04591128-tech/Vive-coding-pdca-system.git
cd Vive-coding-pdca-system

# 2. 仮想環境の作成・有効化
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate

# 3. インストール
pip install -e ".[dev]"

# 4. 環境変数の設定
cp .env.example .env
# .env を編集してAPIキー等を設定

# 5. 動作確認
pytest --tb=short -q --ignore=tests/test_gui.py
```

詳しくは [ローカルセットアップ手順書](../手順書/03_ローカルセットアップ手順書.md) を参照してください。

### Q. GUI は使えますか？

**A.** はい。[Flet](https://flet.dev/) ベースのGUIダッシュボードがあります。

```bash
# GUI依存パッケージも一緒にインストール
pip install -e ".[dev,gui]"

# GUIの起動
vibe-pdca
```

以下の全5画面が利用可能です：ダッシュボード（LLMステータス・コスト・ログ表示）、ゴール入力画面、進捗・PDCA制御画面、介入・ロールバック画面、モード設定画面。ナビゲーションレールで画面を切り替えられます。

## 🔒 セキュリティ

### Q. APIキーはどう管理すればいいですか？

**A.** 以下の方法で安全に管理してください：

1. **`.env` ファイル**に記載（`.gitignore` に登録済みなのでリポジトリには含まれません）
2. **環境変数**として OS に設定
3. **GitHub Secrets** に登録（CI/CD用）

⚠️ **絶対にやってはいけないこと**:
- ソースコードにAPIキーを直接書き込むこと
- `.env` ファイルを Git にコミットすること

### Q. セキュリティの脆弱性を見つけた場合はどうすればいいですか？

**A.** **公開 Issue を作成しないでください。** GitHub の Security Advisory 機能を使って非公開で報告してください。

詳細は [セキュリティポリシー](../管理/セキュリティポリシー.md) を参照してください。

### Q. サンドボックスとは何ですか？

**A.** LLMが生成したコードを**安全に実行する隔離環境**です。Dockerコンテナを使用して、生成コードがシステムに影響を与えないようにしています。

詳細は [ADR-007 サンドボックス設計](../adr/ADR-007_サンドボックス設計.md) を参照してください。

## 🔄 開発・コントリビューション

### Q. 開発に参加するにはどうすればいいですか？

**A.** [コントリビューション説明書](コントリビューション説明書.md) を確認し、以下の手順で参加できます：

1. リポジトリをフォーク
2. Feature ブランチを作成
3. 変更を実装（テスト追加を忘れずに）
4. `ruff check` / `mypy` / `pytest` をパス
5. Pull Request を作成

### Q. テストの書き方のルールはありますか？

**A.** はい。以下のルールに従ってください：

- テストファイルは `tests/test_*.py` の命名規則
- `pytest` を使用
- 非同期テストには `pytest-asyncio` を使用
- 外部API呼び出しはモック（`unittest.mock`）を使用
- テストは独立して実行可能であること

### Q. コーディング規約はありますか？

**A.** `ruff` と `mypy` のルールに従ってください：

| ツール | 設定 |
|--------|------|
| **ruff** | `pyproject.toml` の `[tool.ruff]` セクション。Python 3.12ターゲット、100文字行幅 |
| **mypy** | strict モード。GUI モジュールは除外 |

```bash
# コード品質チェック
ruff check src/ tests/

# 型チェック
mypy src/ --exclude 'src/vibe_pdca/gui/'
```

## 📊 運用

### Q. PDCAサイクルのコストはどのくらいかかりますか？

**A.** 使用するLLMモデルとサイクル回数によります。目安：

| モデル | 1サイクルあたりの概算コスト |
|--------|--------------------------|
| GPT-4o-mini | $0.01〜$0.05 |
| GPT-4o | $0.05〜$0.50 |
| Claude 3.5 Sonnet | $0.03〜$0.30 |
| ローカルLLM（Ollama） | **$0**（電気代のみ） |

コスト上限設定やリアルタイム監視機能で予算管理が可能です。

### Q. 5ペルソナのレビューとは何ですか？

**A.** LLMが**5つの異なる視点**でコードをレビューする仕組みです：

| ペルソナ | 役割 | 確認ポイント |
|---------|------|------------|
| **PM** | プロジェクトマネージャー | 要件充足、スケジュール、リスク |
| **書記** | ドキュメンテーション | コメント、ドキュメント、可読性 |
| **プログラマ** | コード品質 | アルゴリズム、パフォーマンス、保守性 |
| **デザイナ** | 設計・UI | アーキテクチャ、インターフェース |
| **ユーザ** | エンドユーザー視点 | 使いやすさ、エラーメッセージ |

### Q. ログはどこに出力されますか？

**A.** 以下の場所にログが出力されます：

| ログ種類 | 場所 | 説明 |
|---------|------|------|
| 監査ログ | `audit/` | チェーンハッシュ付きの改ざん検知ログ |
| PDCAサイクルログ | メトリクス | コスト・品質スコア等 |
| LLMリクエストログ | 内部 | モデル呼び出しの記録 |
