## 🔰 パソコン初心者の方へ — 基礎知識ガイド

手順書を読み進める前に、以下の基本用語を押さえておきましょう。  
**すでにご存知の方は、次の「手順書一覧と進め方」まで読み飛ばしてOKです。**

### 「ターミナル」「コマンドプロンプト」とは？

パソコンに **文字で命令を入力する画面** のことです。普段マウスでクリックして操作する代わりに、キーボードで命令文を打ち込みます。

- **macOS** → 「ターミナル」アプリ（Launchpad → その他 → ターミナル）
- **Windows** → 「コマンドプロンプト」または「PowerShell」（スタートメニューで「cmd」と検索）
- **Linux（Ubuntu）** → 「端末」アプリ（画面上部のアクティビティで「terminal」と検索）

> 💡 **たとえるなら**: レストランで料理を注文するとき、メニューを指さす（＝マウス操作）か、口頭で料理名を伝える（＝ターミナル操作）かの違いです。ターミナルは「口頭で伝える」方法です。

### 「Ubuntu（ウブントゥ）」とは？

**Linux** と呼ばれるパソコン用の基本ソフト（OS）の一種です。Windows や macOS と同じ役割を持ちます。  
サーバー（24時間動き続けるコンピュータ）で最も広く使われています。

- 無料で利用可能
- このシステムの本番サーバーでは **Ubuntu 22.04 LTS** を使用
- ローカル開発では macOS / Windows でもOKです（Windows の場合は WSL2 で Ubuntu を利用）

### 「WSL2（ダブリューエスエルツー）」とは？

**Windows 上で Ubuntu（Linux）を動かすための仕組み** です。Windows ユーザーが Linux 向けの手順をそのまま使えるようになります。

**WSL2 の導入手順（Windows ユーザー向け）**：
1. スタートメニューで「PowerShell」を検索し、**右クリック → 管理者として実行**
2. 以下のコマンドを入力して Enter キーを押す：
   ```
   wsl --install
   ```
3. パソコンを再起動する
4. 再起動後、自動的に Ubuntu のセットアップ画面が表示される
5. ユーザー名とパスワードを設定する（このパスワードは `sudo` コマンドで使います）

> 💡 WSL2 が正しく動いているか確認するには、PowerShell で `wsl --version` と入力してバージョンが表示されればOKです。

### 「コマンド」の読み方

手順書の中で、このような灰色の枠（コードブロック）が出てきます：

```bash
python3 --version
```

これは **ターミナルに入力する命令文** です。以下のルールで読んでください：

| 記号 | 意味 |
|------|------|
| `$` で始まる行 | ターミナルに入力する行（`$` 自体は入力しない） |
| `#` で始まる行 | 説明コメント（入力不要） |
| `sudo` で始まる | 管理者権限で実行する命令（パスワード入力を求められます） |
| `→` や「期待される出力」 | コマンド実行後に表示されるはずの結果 |

> ⚠️ **注意**: コマンドは **1行ずつ** コピー＆ペーストして Enter キーで実行してください。複数行をまとめて貼り付けると、意図しない動作をすることがあります。

## 📋 手順書一覧と進め方

### 🖥️ ルートA：本番サーバー環境を構築する場合

サーバーを購入するところから始めて、本番稼働まで進めます。

> 💡 **OpenClaw（ハードウェア監視）について**  
> ルートAでは、サーバーのGPU温度・ディスク容量・電源状態などを24時間自動監視する **OpenClaw** を導入します（02 手順書の Phase 8）。  
> OpenClaw はオープンソースの AI エージェントプラットフォームで、本システムではハードウェア監視層として活用します。  
> 詳しくは [OpenClaw説明書](../説明書/OpenClaw説明書.md) を参照してください。

| 順番 | 手順書 | 所要時間 | 内容 |
|:----:|--------|---------|------|
| 01 | [ハードウェア構築手順書](01_ハードウェア構築手順書.md) | 2〜4週間 | サーバー購入→組立→OS導入→GPUドライバー |
| 02 | [システムセットアップ手順書](02_システムセットアップ手順書.md) | 4〜8時間 | Docker→vLLM→GitHub App→Discord→**OpenClaw**→初回稼働 |
| 04 | [運用保守手順書](04_運用保守手順書.md) | 随時参照 | 日常運用→**OpenClaw監視**→障害対応→メンテナンス |

> 💡 **デスクトップPCで運用する場合、OpenClaw は不要です**  
> OpenClaw はサーバーの24時間監視を目的としたツールです。デスクトップPCで開発・テストする場合は導入する必要はありません。
