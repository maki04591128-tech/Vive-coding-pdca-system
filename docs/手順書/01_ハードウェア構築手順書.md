## 6. Phase 4：設置場所の準備

### 6.1 電源の確認

```
家庭用 100V コンセント（15A）
  → 最大 1,500W まで対応
  → 入門構成（～600W）のみ対応可能
  → 推奨構成以上は不足の可能性あり

家庭用 200V コンセント（20A）← 推奨構成以上はこちら
  → 最大 4,000W まで対応
  → 余裕を持って運用可能
  → エアコン用の既存コンセントが流用できる場合あり

⚠️ 200V コンセントの増設は電気工事士の免許が必要です
  → 電気工事業者への依頼が必要（工事費：3〜5万円程度）
```

### 6.2 設置環境の要件

```
【必須条件】
  □ 室温：25℃ 以下（エアコンで維持）
  □ 背面から 30cm 以上の空間（排気用）
  □ 前面に吸気の障害物がないこと
  □ 耐荷重：サーバー本体 20〜30kg を支えられる棚・ラック
  □ 有線 LAN 接続可能（Wi-Fi は推奨しない）

【推奨条件】
  □ サーバー専用のスペース（騒音対策）
  □ 別室・クローゼット・納戸に設置
  □ 床の防振対策（ゴムマット等）
```

### 6.3 騒音の事前理解

サーバーは非常にうるさいです。設置場所は事前によく検討してください。

| サーバータイプ | 騒音レベル | 比較 |
|-------------|---------|------|
| 1U サーバー | 70〜80dB | 掃除機と同程度 |
| **2U サーバー** | **55〜70dB** | エアコン室外機〜掃除機 |
| タワー型 | 45〜60dB | 比較的静か |

### ✅ 確認ポイント

- [ ] 電源の種類（100V / 200V）と容量を確認した
- [ ] 設置場所を決定し、必須条件をすべて満たしている
- [ ] 騒音対策を検討した
- [ ] UPS の設置場所を確保した

## 7. Phase 5：サーバーの開梱と設置

### 7.1 開梱時の注意

```
1. 段ボール開封時、カッターで中の機器を傷つけないこと
2. サーバーは 20〜30kg あるため、2人で持ち上げること
3. 開梱後、以下を確認：
   □ サーバー本体
   □ 電源ケーブル（2本 = 冗長電源用）
   □ レールキット（付属の場合）
   □ 付属品一式（ネジ類・ドライブキャリア等）
```

### 7.2 設置手順

```
Step 1：UPS を設置場所に配置
Step 2：UPS をコンセントに接続し、充電を開始
Step 3：サーバーをラック・棚に設置
Step 4：電源ケーブル 2 本を UPS に接続（冗長電源）
Step 5：LAN ケーブルをルーター/スイッチに接続
Step 6：ディスプレイ（VGA）とキーボード（USB）を接続
```

- [ ] LAN ケーブルが接続されている

- [ ] 背面に 30cm 以上のスペースがある

## 8. Phase 6：GPU の取り付け

> **⚠️ 静電気対策を必ず行ってください！**  
> GPU は静電気で壊れることがあります。静電気防止リストバンドを着用し、サーバーの金属フレームに触れてから作業を始めてください。

### 8.1 取り付け手順

```
Step 1：サーバーの電源ケーブルをすべて抜く
        → 感電防止のため必ず確認

Step 2：サーバーのカバーを開ける
        → ネジ式またはツールレス（レバー式）
        → DELL の場合は背面のラッチを引いてスライド

Step 3：PCIe スロットの位置を確認
        → マニュアルまたはマザーボード上の刻印で x16 スロットを特定
        → GPU は必ず「x16 スロット」に挿すこと

Step 4：スロットのブランクパネル（金属カバー）を外す
        → ネジを外し、パネルを取り除く

Step 5：GPU を PCIe スロットに挿入
        → 端子の向きを確認（金色の端子部分がスロットに入る）
        → まっすぐ均等に力を入れて押し込む
        → 「カチッ」とロック機構が掛かるまで押す

Step 6：GPU をネジで固定
        → ブランクパネルのネジ穴に合わせて固定

Step 7：GPU 補助電源ケーブルを接続（必要な場合）
        → 一部の GPU は 6pin または 8pin の補助電源が必要
        → A100 PCIe はスロットからの給電のみで動作（補助電源不要の場合が多い）

Step 8：2枚目がある場合は同様に取り付け
        → 熱対策のため、隣接スロットではなく 1 スロット空けて配置を推奨

Step 9：カバーを閉じる
```

### ✅ 確認ポイント

- [ ] GPU が PCIe x16 スロットにしっかり挿さっている
- [ ] ネジでブラケットが固定されている
- [ ] 補助電源（必要な場合）が接続されている
- [ ] ケーブルがファンに巻き込まれていない
- [ ] カバーが正しく閉まっている

## 9. Phase 7：メモリ・ストレージの増設

### 9.1 メモリ（RAM）の増設

サーバー購入時にメモリが不足している場合は増設します。

```
サーバー用メモリの注意点：
  ✅ ECC メモリ（Error Correcting Code）を使用すること
  ✅ Registered（RDIMM）タイプであること
  ✅ サーバーの対応世代を確認（DDR4-2666 等）
  ❌ 家庭用PC のメモリ（Non-ECC / Unbuffered）は使用不可

取り付け方：
  1. メモリスロットのラッチ（左右の爪）を開く
  2. メモリの切り欠きの向きを合わせてスロットに差し込む
  3. 「カチッ」と左右のラッチが閉じるまで均等に押す
  4. デュアルチャンネルの場合は対になるスロットに同じ容量を配置
```

### 9.2 NVMe SSD の取り付け

```
取り付け方：
  1. M.2 または U.2 スロットの位置を確認
  2. NVMe SSD を斜めに挿入し、ネジで固定
  3. ホットスワップ対応ベイがある場合はドライブキャリアに固定して挿入
```

### ✅ 確認ポイント

- [ ] メモリが正しいスロットに挿さっている
- [ ] NVMe SSD が取り付けられている
- [ ] すべてのラッチが閉じている

## 10. Phase 8：電源投入とBIOS設定

### 10.1 初回電源投入

```
Step 1：電源ケーブルが UPS に接続されていることを確認
Step 2：UPS の電源を ON にする
Step 3：サーバー前面の電源ボタンを押す
Step 4：ディスプレイにメーカーロゴ（DELL / HP）が表示されることを確認
Step 5：POST（Power-On Self-Test）が完了するまで待つ
        → メモリチェック・デバイス検出が自動で実行される
```

### 10.2 BIOS/UEFI 設定

POST 完了後、以下のキーで BIOS 設定に入ります。

| メーカー | BIOS 起動キー |
|---------|-------------|
| DELL | F2 |
| HP | F9 または F10 |

#### 確認・変更する項目

```
1. Boot Mode：UEFI モード（Legacy ではなく UEFI を選択）
2. PCIe Configuration：
   → GPU が認識されていることを確認
   → PCIe 速度が Gen3 x16 以上であることを確認
3. Memory Configuration：
   → 搭載メモリ容量が正しいことを確認
   → ECC が有効であることを確認
4. Boot Order：
   → USB を最優先に設定（OS インストール用）
5. iDRAC / iLO（リモート管理）：
   → IP アドレスを設定（後でリモートアクセスに使用）
   → 管理者パスワードを設定
6. 設定を保存して再起動
```

### ✅ 確認ポイント

- [ ] サーバーが正常に起動した
- [ ] BIOS で GPU が認識されている
- [ ] メモリ容量が正しく表示されている
- [ ] Boot Mode が UEFI になっている
- [ ] iDRAC / iLO の IP アドレスとパスワードを記録した

## 11. Phase 9：Ubuntu OSのインストール

### 11.1 Ubuntu インストール用 USB の作成

別の PC で以下を実施します。

```
Step 1：Ubuntu 22.04 LTS Server ISO をダウンロード
        → https://releases.ubuntu.com/22.04/
        → 「Ubuntu Server」を選択（Desktop ではなく Server）

Step 2：USB メモリに ISO を書き込む
        → Windows：Rufus（https://rufus.ie/）を使用
        → macOS：balenaEtcher を使用
        → Linux：
           sudo dd if=ubuntu-22.04-live-server-amd64.iso \
                  of=/dev/sdX bs=4M status=progress
```

### 11.2 Ubuntu のインストール手順

```
Step 1：USB メモリをサーバーに挿入し、電源を入れる
Step 2：BIOS の Boot Menu（DELL: F12 / HP: F9）から USB を選択
Step 3：Ubuntu インストーラーが起動

以下の設定を行います：

  言語：English（日本語は後から設定可）
  キーボード：Japanese
  ネットワーク：DHCP 自動取得（後で固定 IP に変更）
  ストレージ：
    → NVMe SSD を選択
    → 「Use an entire disk」を選択（全体を使用）
    → LVM を有効にする
  ユーザー設定：
    → ユーザー名：vibe-admin（例）
    → パスワード：強力なパスワードを設定（16文字以上推奨）
    → サーバー名：vibe-pdca-srv01（例）
  SSH：
    → 「Install OpenSSH server」にチェックを入れる
  追加パッケージ：特に選択不要

Step 4：インストール完了後、USB メモリを抜いて再起動
Step 5：ログイン画面が表示されたら設定したユーザーでログイン
```

### 11.3 初期設定

```bash

# 基本ツールのインストール

```

### 11.4 固定 IP アドレスの設定

```bash
# 現在のネットワーク設定を確認
ip addr show

# Netplan 設定ファイルを編集
sudo vim /etc/netplan/00-installer-config.yaml
```

以下の内容に編集します（IP アドレスは環境に合わせて変更）。

```yaml
network:
  version: 2
  ethernets:
    eno1:                      # ← インターフェース名（ip addr show で確認）
      dhcp4: false
      addresses:
        - 192.168.1.100/24     # ← サーバーの固定 IP
      routes:
        - to: default
          via: 192.168.1.1     # ← デフォルトゲートウェイ（ルーターの IP）

        addresses:
          - 8.8.8.8            # ← DNS サーバー
          - 8.8.4.4
```

```bash

sudo netplan apply

# 接続確認
ping -c 3 google.com
```

- [ ] インターネットに接続できる（`ping google.com`）

## 13. Phase 11：ネットワーク設定とセキュリティ基盤

### 13.1 ファイアウォール（UFW）の設定

```bash
# UFW のインストールと有効化

# デフォルトポリシー：すべての受信を拒否、送信を許可
sudo ufw default deny incoming
sudo ufw default allow outgoing

# SSH を許可（リモート接続用）

# vLLM API ポートを許可（ローカルネットワークのみ）

# Prometheus メトリクスポート（ローカルネットワークのみ）
sudo ufw allow from 192.168.1.0/24 to any port 9090

# Grafana ダッシュボード（ローカルネットワークのみ）
sudo ufw allow from 192.168.1.0/24 to any port 3000

# ファイアウォールを有効化
sudo ufw enable

# 設定の確認
sudo ufw status verbose
```

### 13.2 SSH のセキュリティ強化

```bash
# SSH 設定ファイルを編集
sudo vim /etc/ssh/sshd_config
```

```
# パスワード認証を無効化（公開鍵認証のみ）
PasswordAuthentication no

# root ログインを禁止
PermitRootLogin no

# SSH ポートの変更（任意。デフォルト 22 → 例: 2222）
```

## 15. トラブルシューティング

### Q: サーバーの電源が入らない

```
確認項目：
  1. UPS の電源が ON か
  2. 電源ケーブルが正しく接続されているか
  3. 電源ユニット（PSU）背面のスイッチが ON か（機種による）
  4. コンセントが通電しているか（他の機器で確認）
```

### Q: BIOS で GPU が認識されない

```
確認項目：
  1. GPU が PCIe スロットにしっかり挿さっているか（再度押し込む）
  2. 正しい PCIe x16 スロットに挿しているか
  3. GPU の補助電源（必要な場合）が接続されているか
  4. BIOS で PCIe スロットが無効化されていないか
```

### Q: nvidia-smi で GPU が表示されない

```
確認項目：
  1. NVIDIA ドライバーが正しくインストールされているか
     → sudo dkms status で nvidia が表示されるか
  2. Secure Boot が有効になっていないか
     → BIOS で Secure Boot を無効にする
  3. カーネルバージョンとドライバーの互換性
     → sudo apt install -y nvidia-driver-535 で再インストール
```

### Q: Ubuntu インストール中にストレージが見つからない

```
確認項目：
  1. NVMe SSD が物理的に接続されているか
  2. BIOS でストレージが認識されているか
  3. RAID コントローラーが介在している場合、JBOD モードに変更する
```

### Q: サーバーの騒音がひどい

```
対策：
  1. ファン速度を iDRAC / iLO から制御する（静音プロファイル設定）
  2. 防音ラックボックスを検討
  3. 別室・クローゼットに設置
  4. DELL の場合：ipmitool raw 0x30 0x30 0x01 0x00 で手動制御可
     （自己責任。温度上昇に注意）
```

### Q: GPU 温度が 80℃ を超える

```
対策：
  1. エアフローを確認（前面吸気・背面排気が遮られていないか）
  2. 室温が 25℃ 以下か確認
  3. GPU 間のスペースを空ける（隣接スロットを避ける）
  4. GPU の電力制限を設定する：
     sudo nvidia-smi -pl 200    # ← TDP 250W の GPU を 200W に制限
```

## 16. 用語集

| 用語 | 説明 |
|------|------|
| **BIOS / UEFI** | サーバーの基本入出力システム。ハードウェアの初期設定を行う画面 |
| **POST** | Power-On Self-Test。電源投入時のハードウェア自己診断 |
| **iDRAC** | DELL のリモート管理チップ。電源 OFF でもリモート操作可能 |
| **iLO** | HP のリモート管理チップ。iDRAC と同等の機能 |
| **ECC メモリ** | エラー訂正機能付きメモリ。ビット化けを自動修正する |
| **RDIMM** | Registered DIMM。サーバー向けのメモリ規格 |
| **NVMe** | Non-Volatile Memory Express。高速な SSD の接続規格 |
| **PCIe x16** | GPU を接続する高速スロット。x16 = 16 レーン |
| **UPS** | 無停電電源装置。停電時にバッテリー電源を供給する |
| **UFW** | Uncomplicated Firewall。Ubuntu の標準ファイアウォール |
| **VRAM** | GPU 専用のビデオメモリ。LLM モデルはここに収まる必要がある |
| **CUDA** | NVIDIA の GPU 汎用計算プラットフォーム |
| **テンソル並列** | 1 つのモデルを複数 GPU に分割して推論する技術 |
| **ブロワー型冷却** | 前面→背面に排気するサーバー向け GPU 冷却方式 |
| **SXM** | A100 の特殊コネクタ形式。専用サーバー（DGX 等）でのみ使用可能 |
| **Secure Boot** | OS の改ざんを防ぐセキュリティ機能。GPU ドライバーと干渉する場合がある |
| **Netplan** | Ubuntu のネットワーク設定ツール |
