## 1. 概要

本システムは **2段階のEXE配布方式** を採用しています：

```
┌──────────────────────┐
│  VibePDCA-Setup.exe  │  ← ダウンローダー EXE（軽量、配布用）
│  （ダウンローダー）     │
└──────────┬───────────┘
           │ 実行すると...
           ▼
    1. GitHub Releases からソースをダウンロード
    2. Python 依存パッケージをインストール
    3. 設定ファイル・リソースを展開
    4. メインアプリケーション EXE をビルド
           │
           ▼
┌──────────────────────┐
│    VibePDCA.exe      │  ← メインアプリケーション EXE
│  （PDCA システム）     │
└──────────────────────┘
```

### なぜ2段階？

- **ダウンローダー EXE**: 軽量（数MB）で配布しやすい。常に最新版をダウンロードするため、EXE自体の更新頻度が低い
- **メインアプリ EXE**: 全依存関係を含むため大きい（数十〜数百MB）。ダウンローダーが必要なリソースをすべて取得してからビルドするため、環境依存の問題が少ない

## 2. アーキテクチャ

```
installer/
├── __init__.py              # パッケージ初期化
├── downloader.py            # ダウンローダー本体（CLI / EXE のエントリポイント）
├── build.py                 # ビルドスクリプト（開発者向け）
└── installer_config.yml     # インストーラー設定

ダウンローダー EXE に含まれるもの:
  - downloader.py （メインロジック）
  - installer_config.yml （設定）
  - Python ランタイム（PyInstaller 同梱）

ダウンローダーが行うこと:
  1. GitHub Releases API で最新リリースを確認
  2. ソースコードの tarball をダウンロード
  3. pip install で依存パッケージをインストール
  4. config/, docker/ をインストール先にコピー
  5. PyInstaller でメインアプリ EXE を生成
```

## 3. 前提条件

### ビルド環境（ダウンローダーEXE作成側）

| 項目 | 必要なもの |
|------|-----------|
| OS | Windows 10/11（Windows用EXE作成時）、macOS、Linux |
| Python | 3.12 以上 |
| pip パッケージ | `pyinstaller>=6.0`, `pyyaml>=6.0` |

### エンドユーザー環境（ダウンローダーEXE実行側）

| 項目 | 必要なもの |
|------|-----------|
| OS | Windows 10/11 |
| Python | 3.12 以上（インストール済み、PATHに追加済み）|
| インターネット | GitHub API へのアクセス |
| ディスク | 2GB 以上の空き |

## 4. ダウンローダー EXE のビルド

### 4.1 環境準備

```bash
# リポジトリのクローン
git clone https://github.com/maki04591128-tech/Vive-coding-pdca-system.git
cd Vive-coding-pdca-system

# 仮想環境の作成・有効化
python -m venv .venv
source .venv/bin/activate    # Linux/macOS
# .venv\Scripts\activate     # Windows

# 必要パッケージのインストール
pip install pyinstaller>=6.0 pyyaml>=6.0
```

### 4.2 ビルド実行

```bash
# ダウンローダー EXE のみビルド
python installer/build.py downloader

# メインアプリ EXE のみビルド（ローカルソースから）
python installer/build.py app

# 両方ビルド
python installer/build.py all

# 出力先を指定する場合
python installer/build.py downloader --output-dir ./release
```

### 4.3 出力

```
dist/
├── VibePDCA-Setup.exe    ← 配布するダウンローダー EXE
└── VibePDCA.exe          ← メインアプリケーション EXE（app ビルド時）
```

## 5. メインアプリケーション EXE のビルド

ダウンローダーを使わず直接メインアプリのEXEを作成する場合：

```bash
# プロジェクトルートから
python installer/build.py app
```

または手動で PyInstaller を実行：

```bash
pyinstaller \
  --name VibePDCA \
  --onefile \
  --noconsole \
  --paths src \
  --add-data "config/:config/" \
  --add-data "docker/:docker/" \
  --hidden-import vibe_pdca \
  --hidden-import vibe_pdca.gui.app \
  --hidden-import flet \
  src/vibe_pdca/gui/app.py
```

## 6. インストーラー設定のカスタマイズ

`installer/installer_config.yml` で以下を変更できます：

```yaml
installer:
  # アプリ名
  app_name: "VibePDCA"
  version: "0.1.0"

  # GitHub リリース元
  github:
    owner: "maki04591128-tech"
    repo: "Vive-coding-pdca-system"
    release_tag: "latest"    # "v0.1.0" 等で固定バージョン指定も可能

  # インストール先（プラットフォーム別）
  install:
    default_dir_windows: "%LOCALAPPDATA%\\VibePDCA"
    default_dir_macos: "~/Applications/VibePDCA"
    default_dir_linux: "~/.local/share/VibePDCA"

  # PyInstaller 設定
  pyinstaller:
    onefile: true         # 単一ファイルにするか
    console: false        # コンソールウィンドウを表示するか
    icon: null            # アイコンファイルパス（例: "assets/icon.ico"）
```

## 7. 配布方法

### 7.1 GitHub Releases での配布

1. **リリースの作成**:
   ```bash
   git tag v0.1.0
   git push origin v0.1.0
   ```

2. **GitHub Releases ページ** で新しいリリースを作成

3. **ダウンローダー EXE をアセットとしてアップロード**:
   - `dist/VibePDCA-Setup.exe` をドラッグ＆ドロップ

4. ユーザーは `VibePDCA-Setup.exe` をダウンロードして実行するだけ

### 7.2 ファイル共有での配布

- `VibePDCA-Setup.exe`（通常数MB）を共有するだけ
- メインアプリを直接配布する場合は `VibePDCA.exe` を共有

## 9. トラブルシューティング

### Q: 「Python 3.12 以上が必要です」と表示される

Python 3.12 以上をインストールしてください：
- https://www.python.org/downloads/
- インストール時に「Add Python to PATH」にチェックを入れてください

### Q: ダウンロードが失敗する

- インターネット接続を確認してください
- プロキシ環境の場合、環境変数 `HTTP_PROXY` / `HTTPS_PROXY` を設定してください
- GitHub API のレート制限に達した可能性があります。しばらく待ってから再試行してください

### Q: PyInstaller のビルドが失敗する

```bash
# PyInstaller を最新版に更新
pip install --upgrade pyinstaller

# ウイルス対策ソフトが干渉している可能性があります
# 一時的にリアルタイムスキャンを無効にして再試行してください
```

### Q: EXE 実行時にウイルス対策ソフトが警告する

PyInstaller で生成された EXE は一部のウイルス対策ソフトで誤検知されることがあります：
- ウイルス対策ソフトの例外リストに追加してください
- Windows の場合: Windows Defender → 設定 → 除外 → フォルダの追加
