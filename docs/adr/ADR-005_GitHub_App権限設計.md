# ADR-005：GitHub App 権限設計

**文書番号**：VCS-ADR-005  
**ステータス**：承認済み  
**作成日**：2026-02-27  
**決定者**：実装担当  
**関連要件**：§9.6  
**関連ADR**：ADR-004（GitHub Search APIの認証に本ADRの権限スコープを使用）、ADR-006（リポジトリ単位のインストールが物理分離の前提）

---

## 背景・課題

本システムはGitHubと密に連携し、Issue/PR/CI/マイルストーンの管理を行う。認証方式としてGitHub Appが確定済み（§9.6）であり、権限スコープを最小化しつつ必要な機能を満たす設計が必要である。

---

## 選択肢

### 案A：Personal Access Token（PAT）方式
- **利点**：セットアップが簡単
- **欠点**：ユーザーに紐づくため監査上不利、Organization全体の権限が付与されやすい

### 案B：GitHub App方式（採用）
- **利点**：リポジトリ単位の権限制御が可能、インストールトークンは自動ローテーション、監査ログでアクション主体を識別可能
- **欠点**：セットアップがやや複雑

---

## 採用理由

**案B（GitHub App）を採用する。** リポジトリ単位のインストールにより物理分離要件を満たし、トークンの自動ローテーションで秘密情報管理の負担を軽減できる。

---

## 権限スコープ

| 権限 | スコープ | 用途 |
|------|---------|------|
| Contents | Read & Write | コードのPR作成・ブランチ操作 |
| Issues | Read & Write | Issue生成・ラベル付与・クローズ |
| Pull requests | Read & Write | PRの作成・コメント投稿 |
| Actions | Read | CIステータス収集 |
| Checks | Read & Write | CHECKフェーズのステータス書き込み |
| Metadata | Read | リポジトリ情報取得（必須） |
| Milestones | Read & Write | マイルストーン管理 |
| Workflows | Read | CI設定の読み取り |

---

## 禁止事項

- **Workflows Write は付与しない**：CI設定変更はA操作として人間承認を要する
- **Organization全体への付与は行わない**：リポジトリ単位でインストールする
- **Admin権限は付与しない**：リポジトリの設定変更はスコープ外

---

## 撤回条件

- GitHub App APIに大幅な仕様変更があった場合
- 別の認証方式がより適切な選択肢として成熟した場合
