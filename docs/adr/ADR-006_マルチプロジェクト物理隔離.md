# ADR-006：マルチプロジェクト物理隔離

**文書番号**：VCS-ADR-006  
**ステータス**：承認済み  
**作成日**：2026-02-27  
**決定者**：実装担当  
**関連要件**：§26.1  
**関連ADR**：ADR-004（物理分離によりRAG検索スコープがリポジトリ単位で自動確保）、ADR-005（GitHub Appのリポジトリ単位インストールが本ADRの前提）、ADR-007（サンドボックスもプロジェクト単位で起動・破棄）

---

## 背景・課題

本システムは複数プロジェクトを同時に扱う能力を持つ（§26.1）。プロジェクト間のデータ漏洩・権限逸脱・障害波及を防止するため、隔離方式を決定する必要がある。

---

## 選択肢

### 案A：論理分離方式
- **利点**：リソース効率が高い
- **欠点**：フィルタリングの実装ミスによるデータ漏洩リスク、障害が全プロジェクトに波及

### 案B：物理分離方式（採用）
- **利点**：データ漏洩リスクが構造的に排除、障害の波及範囲が限定、監査での説明が容易
- **欠点**：リソース使用量が増大、セットアップコストが高い

---

## 採用理由

**案B（物理分離）を採用する。** セキュリティ要件と監査要件を考慮し、プロジェクト間のデータ漏洩リスクは構造的に排除すべきである。

---

## 分離対象

| リソース | 分離方式 |
|---------|---------|
| GitHubリポジトリ | プロジェクトごとに独立 |
| LLM APIキー | プロジェクトごとに別キーを発行（共有禁止） |
| GitHub Appインストール | リポジトリ単位（Organization全体付与禁止） |
| 監査ログ保存先 | プロジェクトごとに独立 |
| コスト上限 | プロジェクトごとに独立して設定・監視 |
| Discordチャンネル | プロジェクトごとに専用チャンネル |
| オーケストレータプロセス | プロジェクト単位でプロセス分離（共有プロセス禁止） |

---

## 撤回条件

- プロジェクト数が大規模（50以上）に達し、リソースコストが許容範囲を超えた場合
- ただし、APIキー・ログ・状態の物理分離は維持する
