## 📖 この文書について

> この文書は、10サイクルごとの統括レビュー時にサブPM/書記が生成する**失敗パターン抽出結果のテンプレート**です。  
> たとえるなら、「病院のカルテのひな形」のようなものです。過去の不具合パターンを記録・分析し、次のサイクルに活かすためのフォーマットを定義しています。

### 🔰 この文書で使われる主要な用語

| 用語 | かんたんな説明 |
|------|-------------|
| **サイクル** | PLAN→DO→CHECK→ACTの1周のこと |
| **統括レビュー** | 10サイクルごとに行う振り返り・総まとめ |
| **blocker** | 進行を妨げる重大な指摘事項 |
| **PLANプロンプト** | AIに指示を出すための入力テキスト。改善内容を追記して次サイクルに反映する |
| **B操作** | 中程度の影響がある操作。ペルソナ3承認が必要 |
| **サブPM/書記** | AIペルソナの役割。レビュー結果の集約を担当 |

## 概要

本テンプレートは10サイクルごとの統括レビュー時にサブPM/書記が生成する「失敗パターン抽出結果」のフォーマットを定義する。抽出された失敗パターンは次サイクル以降のPLANプロンプトへ注意事項として自動追記される。

## 運用ルール

### 生成タイミング

- 10サイクルごとの統括レビュー時にサブPM/書記が自動生成する
- 統括レビュー要約（VCS-SRTPL-001）と同時に生成する

### 抽出対象

| 対象 | 判定基準 |
|------|---------|
| CI連続失敗パターン | 同一原因のCI失敗が3回以上発生 |
| blocker指摘の再発 | 同一カテゴリのblockerが2サイクル以上で検出 |
| スコープ逸脱の傾向 | スコープ逸脱がACTで2回以上記録 |
| リトライ多発箇所 | 同一タスクでリトライが2回以上発生 |

### プロンプト追記の承認

- フィードバックによるプロンプト変更はB操作（ペルソナ3承認）として扱う
- 承認後、次サイクルのPLANプロンプトに自動追記される
- 追記内容はGitHub上で版管理される
