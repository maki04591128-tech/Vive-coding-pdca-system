## 📖 この文書について

> この文書は、[テスト・デバッグ検査要綱](../管理/テスト・デバッグ検査要綱.md) に基づいてローカル検査を実施した結果を記録するための**報告テンプレート**です。  
> たとえるなら、「健康診断の結果用紙」のようなものです。各検査項目の結果を記録し、全体の合否を判定します。

### 🔰 この文書で使われる主要な用語

| 用語 | かんたんな説明 |
|------|-------------|
| **PASS** | 検査に合格。期待通りの結果が得られた |
| **FAIL** | 検査に不合格。期待と異なる結果が出た |
| **SKIP** | 検査を実施しなかった（理由を備考に記載） |
| **N/A** | この環境では該当しない検査項目 |
| **リグレッション** | 以前は正しく動いていた機能が壊れること |

> 📌 **関連ドキュメント**  
> - 検査手順の詳細 → [テスト・デバッグ検査要綱](../管理/テスト・デバッグ検査要綱.md)  
> - CIパイプライン → [.github/workflows/ci.yml](../../.github/workflows/ci.yml)  
> - 変更履歴 → [変更履歴](../管理/変更履歴.md)

## 使い方

1. 上記テンプレート（` ``` ` で囲まれた部分）をコピーして新しいファイルに貼り付ける
2. ファイル名は `検査結果報告_YYYYMMDD_担当者名.md` とする
3. [テスト・デバッグ検査要綱](../管理/テスト・デバッグ検査要綱.md) の手順に従って各検査を実施
4. 結果を各欄に記入（PASS / FAIL / SKIP / N/A）
5. FAILがある場合は「不具合一覧」に詳細を記載
6. `<details>` ブロック内にコマンド出力のコピーを貼り付け
7. 「検査完了確認」のチェックボックスを埋めて完了

### クイック検査の場合

T1〜T3 の結果と T4-1（全テスト一括）の結果のみ記入し、その他は `SKIP（クイック検査のため）` と記入してください。

## 運用ルール

| ルール | 詳細 |
|--------|------|
| **保管場所** | `docs/検査結果/` ディレクトリ（必要に応じて作成） |
| **保管期間** | リリースから6ヶ月間 |
| **レビュー** | 検査結果は Maintainer 以上がレビューする |
| **再検査基準** | 不具合修正後は、該当する検査カテゴリ全体を再実施 |
