## 脆弱性の報告

セキュリティ上の脆弱性を発見した場合は、**公開 Issue を作成せず**、以下の手順で報告してください。

1. **GitHub Security Advisory** を使用して非公開で報告する
   - リポジトリの「Security」タブ → 「Report a vulnerability」から報告
2. 報告には以下を含めてください：
   - 脆弱性の概要と再現手順
   - 影響範囲の評価（該当モジュール、影響を受けるデータ）
   - 可能であれば修正案

## 対応方針

- 報告受領後 **48時間以内** に初期対応を行います
- 重大度に応じた対応スケジュール：
  - **Critical / High**: 72時間以内に修正パッチをリリース
  - **Medium**: 次回リリースサイクルで修正
  - **Low**: バックログに追加し優先度に応じて対応

## セキュリティ設計の概要

本システムでは以下のセキュリティ対策を実装しています：

- **監査ログ**: チェーンハッシュによる改ざん検知（`audit/`）
- **RBAC**: Owner/Maintainer/Viewer のロールベースアクセス制御（`governance/`）
- **入力バリデーション**: Pydantic スキーマによる厳密な入力検証（`models/`）
- **サンドボックス実行**: Docker コンテナによる隔離実行環境（`docker/`）
- **シークレット管理**: 環境変数ベースの秘密情報管理（`.env.example` 参照）
- **依存関係スキャン**: CI パイプラインでの自動脆弱性検出
- **A/B/C 操作分類**: リスクレベルに応じたガバナンス承認ワークフロー
