## 脆弱性の報告

セキュリティ上の脆弱性を発見した場合は、**公開 Issue を作成せず**、以下の手順で報告してください。

1. **GitHub Security Advisory** を使用して非公開で報告する
   - リポジトリの「Security」タブ → 「Report a vulnerability」から報告
2. 報告には以下を含めてください：
   - 脆弱性の概要と再現手順
   - 影響範囲の評価（該当モジュール、影響を受けるデータ）
   - 可能であれば修正案

## 対応方針

- 報告受領後 **48時間以内** に初期対応を行います
- 重大度に応じた対応スケジュール：
  - **Critical / High**: 72時間以内に修正パッチをリリース
  - **Medium**: 次回リリースサイクルで修正
  - **Low**: バックログに追加し優先度に応じて対応

## セキュリティ設計の概要

本システムでは以下のセキュリティ対策を実装しています：

- **監査ログ**: チェーンハッシュによる改ざん検知（`audit/`）
- **RBAC**: Owner/Maintainer/Reviewer/Auditor のロールベースアクセス制御（`governance/`）
- **入力バリデーション**: Pydantic スキーマによる厳密な入力検証（`models/`）
- **サンドボックス実行**: Docker コンテナによる隔離実行環境（`docker/`）
- **シークレット管理**: 環境変数ベースの秘密情報管理（`.env.example` 参照）
- **依存関係スキャン**: CI パイプラインでの自動脆弱性検出
- **A/B/C 操作分類**: リスクレベルに応じたガバナンス承認ワークフロー

## OpenClaw セキュリティ境界（🖥️ サーバー運用専用）

> デスクトップPC運用の場合、このセクションは該当しません。

サーバー運用時に導入するハードウェア監視層（OpenClaw）のセキュリティ設計：

| 対策 | 詳細 |
|------|------|
| **専用ユーザー分離** | `openclaw-monitor` ユーザー（ログイン不可・`/usr/sbin/nologin`）で実行。PDCA ユーザーとは完全分離 |
| **コマンドホワイトリスト** | `allowed_commands.yml` に記載されたコマンドのみ実行可能（nvidia-smi, df, free 等の読み取り専用コマンド） |
| **ファイルアクセス拒否** | PDCA のソースコード（`/home/vibe-pdca/`）、設定ファイル（`/etc/vibe-pdca/`）、`.env` ファイルへのアクセスを拒否 |
| **ネットワーク分離** | PDCA Web UI（:8080）、データベース（:5432）へのアクセスを拒否 |
| **一方向通信** | Discord Webhook による通知のみ（外部→OpenClaw の通信なし） |
| **自動修復の制限** | 許可される自動操作は 3 種類のみ：GPU ファン制御（85℃超）、vLLM 再起動（3回失敗時）、古いログのアーカイブ（ディスク 90%超） |

詳細は [OpenClaw説明書 §3](../説明書/OpenClaw説明書.md) および [三層独立運用体制アーキテクチャ設計書](../設計書/三層独立運用体制_アーキテクチャ設計書.md) を参照してください。
