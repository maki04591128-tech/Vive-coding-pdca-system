#### CI/CD連携テストに必要なリポジトリ情報

CI Adapter の `get_status()` を実際のCI/CDサービスに接続してテストするには、
以下の情報が必要です（現在はモックテストのみで動作確認済み）。

**GitHub Actions の場合:**

| 項目 | 説明 | 例 |
|------|------|-----|
| **リポジトリオーナー名** | GitHub のユーザー名またはOrg名 | `maki04591128-tech` |
| **リポジトリ名** | 対象リポジトリの名前 | `Vive-coding-pdca-system` |
| **GITHUB_TOKEN** | GitHub Personal Access Token（`repo` スコープ） | `ghp_xxxxxxxxxxxx` |

※ コード内では `GitHubActionsAdapter(owner="...", repo="...")` の形式で指定します。

**GitLab CI の場合:**

| 項目 | 説明 | 例 |
|------|------|-----|
| **プロジェクトID** | GitLab プロジェクトの数値ID | `12345678` |
| **GITLAB_TOKEN** | GitLab Personal Access Token（`api` スコープ） | `glpat-xxxxxxxxxxxx` |

※ GitLab CI を使用しない場合は不要です。

**利用方法:**
```python
# GitHub Actions
adapter = GitHubActionsAdapter(
    owner="maki04591128-tech",
    repo="Vive-coding-pdca-system",
    token="ghp_...",  # または環境変数 GITHUB_TOKEN で設定
)
result = adapter.get_status(run_id="12345")

# GitLab CI
adapter = GitLabCIAdapter(
    project_id="12345678",
    token="glpat-...",  # または環境変数 GITLAB_TOKEN で設定
)
result = adapter.get_status(pipeline_id="999")
```

**現在のテスト状況:** API呼び出しのモックテスト（11件）は全て通過済み。
実際のAPIとの結合テストは、上記の情報が揃い次第実施可能です。
