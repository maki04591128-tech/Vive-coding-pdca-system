### CTR-1【Critical】Discordチャンネル名の不一致（複数ファイル横断）

Discordチャンネル設計書（正式設計）と他文書で使用されるチャンネル名が乖離している。

| ファイル | 使用しているチャンネル名 |
|---------|----------------------|
| `docs/設計書/Discordチャンネル設計書.md` §1（**正式設計**） | `pdca-critical`, `pdca-warning`, `pdca-info`, `pdca-status`, `pdca-cycles`, `pdca-artifacts` |
| `docs/運用/ランブック_通常運用.md` §2.2, §4, §6 | `#pdca-notifications`, `#pdca-approvals` |
| `docs/設計書/三層独立運用体制_アーキテクチャ設計書.md` §1.3 | `#pdca-incidents`, `#pdca-notifications`, `#pdca-approvals` |
| `docs/運用/フォールバック運用手順書.md` §3.3 | `#pdca-notifications` |

**具体的な記述の引用**：

- ランブック §2.2:
  > - [ ] Discord `#pdca-notifications` に起動通知が届いていること
- ランブック §6:
  > Discord `#pdca-approvals` に届く承認要求に YES / NO / HOLD で回答する。
- 三層独立運用体制設計書 §1.3:
  > ```
  > └── 【PDCAシステムチャンネル】（既存）
  >     ├── #pdca-incidents       → P0/P1インシデント（§19）
  >     ├── #pdca-notifications   → P2警告・サイクル完了・B操作通知
  >     └── #pdca-approvals       → A操作承認要求（4/4承認）
  > ```

**問題点**: Discordチャンネル設計書で正式に定義されたチャンネル名（`pdca-critical`, `pdca-warning` 等）と、他文書で使われる旧チャンネル名（`#pdca-notifications`, `#pdca-approvals`, `#pdca-incidents`）が一致しない。運用者がどちらに従うべきか判断できない。

**HW系チャンネルでも同様**：三層設計書は `#hw-incidents`, `#hw-maintenance`, `#hw-status` を使用するが、Discord設計書は `hw-critical`, `hw-warning`, `hw-info`, `hw-status` を定義しており、`#hw-incidents` と `#hw-maintenance` は存在しない。

## 3. 統合すべきドキュメントペア

### MERGE-1【推奨】ランブック + 運用保守手順書

| ファイル | 内容 |
|---------|------|
| `docs/運用/ランブック_通常運用.md` | 起動・停止・再開・承認・モード切替・ドライラン・コスト設定・日常チェック |
| `docs/手順書/運用保守手順書.md` | 日常運用・監視・障害対応・メンテナンス |

**理由**: 両文書とも「日常の運用操作」をカバーしており、対象読者も同一（Owner / Maintainer）。運用保守手順書はランブックの上位互換的な内容を持つため、ランブックの独自コンテンツを運用保守手順書に統合し、ランブックは運用保守手順書への参照リダイレクトとするのが合理的。

## 6. 残タスク一覧（`docs/管理/残タスク一覧.md`）の評価

### 文書としての意義

**結論: 引き続き有効な管理文書である。**

理由:
1. **未完了タスクが残存している**: Q1（GUI画面追加）は保留中、Q3（Docker統合テスト / インストーラービルド検証）は中優先度で後回し、Q4（CI/CDリポジトリ設定）は情報待ちのまま。「全タスク完了」の状態ではない
2. **完了済みタスクの記録**として価値がある（何が対応済みで何がペンディングかが明確）
3. **テスト数の内部矛盾**がある: 概要で「1574件」、サマリで「1573件パス / 1件スキップ」とあり、1574 - 1 = 1573 と解釈すれば整合するが、概要の「1574件のテストが**全て**パス」という記述は「1件スキップ」と矛盾する

- 概要の「1574件のテストが全てパス」→「1573件パス / 1件スキップ（GUI）」に修正すべき
- Q1/Q3/Q4 の最終更新日が 2026-03-05 のまま。定期的なステータス更新ルールを設けるべき
