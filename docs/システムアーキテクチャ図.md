## 2. データフロー

### 2.1 PDCAサイクルのデータフロー

```
最終到達点（ユーザー入力）
    │
    ↓
[PLAN] ── 入力：最終到達点 / 現マイルストーン / CHECK結果
    │       出力：タスク（最大7件）/ DoD / リスク / 依存関係
    │       実行：PM（モデル：ADR-001参照）
    │       記録先：GitHub Issue
    ↓
[DO]  ── 入力：PLANのタスク
    │       出力：コード差分 / テスト / ドキュメント更新
    │       実行：Claude（Anthropic API）+ サンドボックス
    │       記録先：GitHub PR
    ↓
[CHECK] ── 入力：PR差分 / CI結果 / DoD
    │         出力：統合レビューサマリ / DoD達成判定
    │         実行：5ペルソナ（PM/書記/プログラマ/デザイナ/ユーザ）
    │         記録先：GitHub PRコメント + リポジトリ内JSON
    ↓
[ACT]  ── 入力：統合レビュー / DoD判定 / 失敗履歴
    │        出力：採否 / 次サイクル方針 / 進捗更新
    │        実行：PM（モデル：ADR-001参照）
    │        記録先：GitHub Issue + 決定ログ
    ↓
  DoD達成？ ─── Yes → マイルストーン更新 → 最終到達点達成？ → 完了報告
    │
    No → 次サイクルPLANへ
```

### 2.2 承認フロー（A操作）

```
オーケストレータ → A操作検出 → 承認要求生成（§17.4テンプレ）
    ↓
Discord通知 → 4ロール承認待ち（12h タイムアウト）
    ↓
  ┌── 4/4承認 → 実行 → 監査ログ記録
  ├── NO → 理由参照 → 別案提示 → ACT記録
  └── タイムアウト / 通知失敗 → 保留
```

### 2.3 コンテキスト管理サイクル

```
サイクル 1〜9  ：通常PDCA（GitHub RAGでドキュメント参照）
サイクル 10   ：統括レビュー + 要約ドキュメント生成（書記）→ GitHub保存
サイクル 11〜99：上記繰り返し（10サイクルごとに統括レビュー）
サイクル 100  ：コンテキストリセット + GitHubドキュメント参照で再構築
```

## 3. コンポーネント間通信

| 発信元 | 宛先 | プロトコル | 内容 |
|--------|------|-----------|------|
| オーケストレータ | LLMゲートウェイ | 内部関数呼び出し | プロンプト実行・レスポンス取得 |
| LLMゲートウェイ | 各LLM API | HTTPS | モデル呼び出し |
| LLMゲートウェイ | ローカルLLM | HTTP（localhost） | フォールバック時のモデル呼び出し |
| オーケストレータ | GitHub API | HTTPS（GitHub App認証） | Issue/PR/Milestone操作 |
| オーケストレータ | サンドボックス | Docker API | コンテナ起動・実行・破棄 |
| オーケストレータ | Discord API | HTTPS | 通知・承認要求 |
| サンドボックス | パッケージレジストリ | HTTPS（ホワイトリスト） | 依存解決 |
| Web UI | オーケストレータ | HTTP/WebSocket | 操作・状態取得 |

## 4. セキュリティ境界

```
┌─ 信頼境界 ──────────────────────────────────────────┐
│  オーケストレータ / LLMゲートウェイ / Web UI          │
│  （すべてのデータを制御下で処理）                      │
└─────────────────────────────────────────────────────┘
         ↕ 認証付き通信
┌─ 半信頼境界 ────────────────────────────────────────┐
│  GitHub API / LLM API / Discord API                   │
│  （認証済みの外部サービス）                            │
└─────────────────────────────────────────────────────┘
         ↕ ホワイトリスト制御
┌─ 不信境界 ──────────────────────────────────────────┐
│  サンドボックス内のコード実行                          │
│  Issue/PR/README等の外部入力テキスト                   │
│  （すべて不信入力として処理）                          │
└─────────────────────────────────────────────────────┘
```
